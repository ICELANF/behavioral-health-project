## 三、对话触发时的知识调用链路

### 3.1 两条调用路径

#### 路径 A：REST 聊天接口（含自动RAG）

```
POST /api/v1/chat/sessions/{id}/messages
         │
         ▼
chat_rest_api.py:send_message()
         │
         ├─ 1. 安全过滤（输入）SafetyPipeline.process_input()
         ├─ 2. 保存用户消息 → ChatMessage
         ├─ 3. 读取历史（最近10条）
         │
         ├─ 4. ▶ RAG增强 ◀
         │       rag_enhance(db, query, agent_id, base_prompt)
         │         │
         │         └─ KnowledgeRetriever.retrieve()
         │               │
         │               ├─ embed_query(query)   [Ollama 1024维]
         │               ├─ SQL: SELECT chunks WHERE scope IN(...)
         │               ├─ 计算余弦相似度 + scope_boost
         │               └─ 返回 RAGContext (top-5 chunks + prompt注入)
         │
         ├─ 5. 构建增强系统提示词（3层知识块注入）
         ├─ 6. 调用 Ollama LLM（qwen3-coder）
         ├─ 7. 提取引用标记 [1][2][3]
         ├─ 8. record_citations() → knowledge_citations表
         ├─ 9. 安全过滤（输出）
         └─ 10. 返回 {user_message, assistant_message, rag:{...}}
```

#### 路径 B：v3 Coach 对话接口（含意图分类）

```
POST /api/v3/chat/message
         │
         ▼
routers/chat.py → CoachAgent.chat()
         │
         ├─ 1. 意图分类（轻量模型）→ intent
         │
         ├─ 2. 判断是否需要RAG:
         │       RAG_INTENTS = {knowledge_qa, strategy_explain,
         │                      coach_dialogue, stage_assessment,
         │                      crisis_support, progress_summary}
         │
         ├─ 3a. [RAG意图] → RAGPipeline.query() 或 coach_query()
         │         │
         │         └─ Qdrant搜索 → context → LLM
         │
         ├─ 3b. [非RAG意图] → LLMRouter直接路由
         │
         └─ 返回 ChatResponse{answer, intent, sources, ...}
```

#### 专用接口

| 接口 | 功能 | RAG方式 |
|------|------|---------|
| `POST /api/v3/chat/knowledge` | 纯知识问答 | RAGPipeline.query() |
| `POST /api/v3/chat/knowledge/search` | 向量检索（无LLM） | Qdrant直搜 |
| `POST /api/v3/chat/prescription` | 行为处方生成 | RAGPipeline.prescription_query() |
