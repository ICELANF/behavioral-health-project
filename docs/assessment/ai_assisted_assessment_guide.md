```
┌─────────────────────────────────────────────────────────────────┐
│                    隐性评估 → 四层诊断 → 行为处方                  │
└─────────────────────────────────────────────────────────────────┘

【数据采集】                    【AI分析】                    【输出应用】
                                    
用户日常对话 ──┐                                             ┌── C端: 正向引导语
任务打卡记录 ──┼──→ 隐性评估引擎 ──→ 四层诊断生成 ──→ ────┼── B端: 完整诊断+处方
CGM/HRV数据  ──┤       ↓                    ↓           │── 专家: 审核确认

                       ↓                    ↓
                  风险预警 ──────→ 教练跟进建议
```

## 二、各层评估的数据来源与置信度

### 第一层：行为诊断

| 诊断维度 | 隐性数据源 | 置信度计算 | 需要补充时机 |
|---------|-----------|-----------|-------------|
| 表层问题 | 主诉分析、指标异常 | 高(0.9) | 很少需要 |
| 中层困难 | 任务失败模式、对话分析 | 中(0.7) | 置信度<0.7 |
| 深层目的 | 长期目标陈述、价值观提取 | 中(0.6) | 建议验证 |
| 内在驱动力 | 目标陈述、主动性行为 | 中(0.65) | 置信度<0.6 |
| 外在事件 | 触发事件日志 | 高(0.85) | 很少需要 |
| 情绪体验 | HRV+文本情感 | 高(0.8) | 很少需要 |
| 认知知识 | 问答表现、理解度 | 中(0.7) | 置信度<0.6 |
| 能力资源 | 任务完成率、时间分配 | 中(0.75) | 置信度<0.7 |
| 社会支持 | 互动记录、提及频率 | 中(0.6) | 建议验证 |
| 心理层次 | 阻抗模式、接受信号 | 中(0.7) | 必须验证 |

### 第二层：SPI评估

| 问卷部分 | 隐性覆盖率 | 自适应策略 |
|---------|-----------|-----------|
| 第一部分(25题) | 约60% | 只问置信度<0.7的维度 |
| 第二部分(20题) | 约50% | 关键问题必问 |
| 第三部分(5题) | 0% | 必须全部填写(主观感受) |

**典型自适应问卷**: 原50题 → 15-25题 (节省50%时间)

## 三、教练工作流程

### 3.1 新学员入组流程

```
Day 1-2: 隐性数据积累
         ↓
Day 3: AI生成第一层诊断初稿 (置信度标注)
         ↓
教练审核 ──→ 确认/修正 ──→ 生成自适应SPI问卷
         ↓
引导用户完成问卷 (15-25题, 约10分钟)
         ↓
AI计算SPI + 生成处方建议
         ↓
教练修改确认处方 ──→ (高风险/敏感案例)专家审核
         ↓
处方生效，进入陪跑阶段
```

| AI建议类型 | 建议来源 | 教练操作 | 修改比例参考 |
|-----------|---------|---------|-------------|

## 四、专家督导要点

### 4.1 必须专家审核的场景

1. **高风险预警 (L1-L2)**
   - AI判定存在危机倾向
   - 连续失联超过3天
   - 明确表达放弃意愿

2. **阶段退行**
   - 从L4/L5退回L3以下
   - SPI下降超过15分

3. **敏感案例**
   - 涉及心理健康问题
   - 存在家庭冲突因素
   - 有自伤风险信号

4. **处方重大调整**
   - 目标难度大幅降低
   - 干预策略根本性改变

### 4.2 审核决策框架

```
收到审核请求
      ↓
查看完整四层诊断 + 历史轨迹
      ↓
评估AI判定准确性
      ↓
┌─────────────────────────────────────┐
│  AI判定准确?                         │
│  ├── 是 → 确认 + 补充指导意见        │
│  └── 否 → 修正判定 + 说明修正理由    │
└─────────────────────────────────────┘
      ↓
决定是否直接介入
      ↓
记录专家意见 → 反馈教练
```

## 六、常见问题处理

### Q1: 隐性评估数据不足怎么办?

**场景**: 新用户入组,对话少,数据积累不够

**处理**:
1. 延长观察期到5-7天
2. 主动发起结构化对话引导
3. 切换到完整版问卷(50题)
4. 标记"数据不足,需持续观察"

### Q2: AI诊断与教练判断不一致?

**处理**:
1. 查看AI判断依据(数据来源+置信度)
2. 如果教练判断更准确 → 修正并记录原因
3. 如果存疑 → 提交专家复核
4. 修正记录用于AI模型优化

### Q3: 用户拒绝填写问卷?

**处理**:
1. 切换到纯隐性评估模式
2. 适当降低诊断置信度
3. 通过对话间接获取关键信息
4. 在跟进中逐步补充验证

### Q4: SPI分数与实际表现不符?

**可能原因**:
- 问卷回答存在社会称许性
- 隐性数据采集周期不足
- 用户状态近期发生变化

**处理**:
1. 对比隐性数据与问卷回答
2. 交叉验证关键维度
3. 缩短复评周期(2周)
4. 加强行为数据监测

   └─────────────────────────┘
         ↓
执行对应预案 → 记录处理过程 → 复盘优化
