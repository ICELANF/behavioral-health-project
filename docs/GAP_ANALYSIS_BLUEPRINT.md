## 一、架构对照：当前 vs 蓝本

### 1.1 架构框架对比

| 维度 | 当前架构 (八爪鱼) | 蓝本架构 | 差异说明 |
|------|-------------------|----------|----------|
| **主线划分** | 3条触手 | 4条主线 | 缺少"陪伴追踪主线" |
| **层次划分** | 5层 (L0-L4) | 5层 (L1-L5) | 命名不同，内涵有差异 |
| **核心定位** | AI健康助手平台 | 行为改变操作系统(BCOS) | 蓝本定位更高 |
| **护城河** | 双引擎AI | 双中枢+工作流+知识沉淀 | 蓝本更强调系统能力 |

### 1.2 主线映射

```
蓝本四主线                    当前三触手
─────────────────────────────────────────────
A. 行为评估主线    ──────▶   (未独立)
B. 行为干预主线    ──────▶   触手1: 专家Chatflow ✅
C. 陪伴追踪主线    ──────▶   触手2: 用户行为养成 ⚠️
D. 成长进化主线    ──────▶   触手3: 教练培养 ⚠️
```

**关键发现**: 蓝本将"行为评估"独立为第一主线(入口系统)，当前架构未体现。

> ❗ D × L2 / L3（成长阶段与能力模型）

## 四、数据体系差距

### 蓝本要求的三类数据

| 数据类型 | 蓝本要求 | 当前状态 |
|----------|----------|----------|
| **A类: 行为数据** | 饮食/运动/睡眠/任务执行/打卡 | ❌ 无采集系统 |

| 层级 | 蓝本要求 | 当前状态 |
|------|----------|----------|
| L1 数据采集层 | 用户输入/设备接口/系统日志 | ❌ |
| L2 数据清洗层 | 时间对齐/单位统一/异常剔除 | ❌ |
| L3 指标画像层 | 依从性指数/风险指数/Trigger Tag | ❌ |
| L4 决策支持层 | 风险预警/干预升级/策略调整 | ❌ |
| L5 学习进化层 | 路径效果评估/最优干预挖掘 | ❌ |

**结论**: 数据体系完全未建设。

## 六、开发治理原则对照

### 蓝本四大原则 vs 当前实践

| 原则 | 蓝本要求 | 当前实践 | 评估 |
|------|----------|----------|------|
| **原则1**: 先挂主线再立模块 | 每个功能必须标注主线+层级 | 按触手划分 | ⚠️ 需细化 |
| **原则2**: 先建母库再做应用 | 模型→动作→Prompt→页面 | 直接做页面 | ❌ 违反 |
| **原则3**: 能力组件化 | 评估/分流/干预/追踪组件 | 未组件化 | ❌ 违反 |
| **原则4**: 设立架构官 | 专人负责结构/术语/重复 | 无此角色 | ❌ 缺失 |

### 蓝本开发顺序 vs 当前顺序

```
蓝本要求顺序:
L1 理念模型 → L2 评估分流 → L3 干预教练 → L4 执行追踪 → L5 进化学习
     ↓              ↓              ↓              ↓
   先冻结        系统大脑       核心引擎        最后做页面

当前实际顺序:
L4 页面(管理后台) → L3 部分(Dify Chatflow) → L1 部分(文档)
     ↓                    ↓                      ↓
   先做页面            部分干预能力           补写文档

❌ 顺序颠倒
```

## 八、调整建议

### 8.1 架构层面建议

#### 建议1: 补齐"行为评估主线"

```
当前:
  触手1(专家) + 触手2(用户) + 触手3(教练)

建议调整为:
  主线A(评估) + 主线B(干预) + 主线C(陪伴) + 主线D(成长)
       ↓            ↓            ↓            ↓
    入口系统      核心引擎      闭环系统      长期价值
```

#### 建议2: 优先建设L2层(系统大脑)

```
优先级调整:

当前:  L4页面 > L3干预 > L1模型 > L2评估 > L5进化
       ─────────────────────────────────────────
建议:  L2评估 > L3干预 > L4页面 > L1模型 > L5进化
         ↑
       最高优先级
```

## 九、风险提示

### 当前最大风险

| 风险 | 说明 | 影响 |
|------|------|------|
| **L2层缺失** | 没有评估分流引擎 | 无法实现个性化干预 |
| **先做页面** | 违反蓝本开发顺序 | 可能需要重构 |
| **术语不统一** | 各模块概念不一致 | 难以集成 |
| **无架构治理** | 没有人守护结构 | 系统可能再次混乱 |

### 蓝本警告

> "每一块都像对的，但拼不起来"

当前风险正是如此：
- 管理后台做得不错 → 但和评估系统怎么连？
- 专家Chatflow做得不错 → 但和用户画像怎么关联？
- 教练培训做得不错 → 但教练动作库在哪里？
