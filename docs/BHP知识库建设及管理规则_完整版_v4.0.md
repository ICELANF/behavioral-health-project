# 第零篇 平台最高层目标（v4.0核心新增）

## 0. 平台三层价值架构

### 0.1 一句话定位

> **行健平台的最高层目标是：通过行为健康解决方案，帮助用户解决痛点问题、完成环境适应，并最终实现自我发现、自我实现、自我超越——成为生命的主人。**
>
> 慢性病管理是入口，不是目的。
> 行为改变是路径，不是终点。
> 最高目标是人的成长与生命意义的实现。

---

### 0.2 三层价值架构图

```
                    ╔══════════════════════════════════════╗
                    ║         第三层：成长超越              ║  ← 平台最高层目标
                    ║   发现自我 → 实现自我 → 超越自我       ║
                    ║       成为生命的主人（S5-S6）          ║
                    ╚══════════════════════════════════════╝
                              ↑ 用户留存动力：
                              内在价值驱动 · 生命意义 · 身份认同
                    
               ╔══════════════════════════════════════════════╗
               ║              第二层：环境适应                 ║
               ║   新习惯融入生活 · 社会支持建立 · 自我效能感    ║
               ║              （S3-S4-S5过渡）                 ║
               ╚══════════════════════════════════════════════╝
                              ↑ 用户留存动力：
                              效能感 · 自主感 · 习惯惯性
               
          ╔══════════════════════════════════════════════════════╗
          ║                  第一层：痛点解决                     ║
          ║     慢性病管理 · 症状缓解 · 危险因素控制              ║
          ║              （S0-S1-S2-S3）                         ║
          ╚══════════════════════════════════════════════════════╝
                              ↑ 用户留存动力：
                              恐惧驱动 · 痛苦驱动 · 医嘱遵从
```

---

### 0.3 三层价值的核心洞见

**为什么三层都需要，缺一不可：**

| 层级 | 如果缺失 | 结果 |
|------|---------|------|
| **痛点层缺失** | 无法触达用户，因为没有"理由使用" | 没有入口，平台没有用户 |
| **适应层缺失** | 痛点缓解后用户流失，行为无法持续 | 用户用了一次就离开 |
| **成长层缺失** | S4→S5的玻璃天花板，平台无法创造真正的生命价值 | 平台永远只是健康管理App，不是生命陪伴工具 |

**S5-S6用户的本质差异**：

```
S0-S4用户问的问题是：
  "我的血糖怎么降？"
  "我怎么减10斤？"
  "我怎么戒掉宵夜？"
  → 问题导向，外部目标驱动

S5-S6用户问的问题是：
  "我想成为一个什么样的人？"
  "这样的生活有什么意义？"
  "我怎么让这种状态持续下去，不只是为了血糖？"
  → 身份认同导向，内在价值驱动
```

**关键转折点（S4→S5）**：

> 这个转折点不是血糖达标了，不是体重降到目标了。
> 转折点是用户开始**用"我是谁"而不是"我有什么问题"来定义自己**。
> 
> S4的自我叙事："我是一个努力控制糖尿病的人。"
> S5的自我叙事："我是一个重视健康、热爱生活的人，糖尿病是我人生故事的一部分。"
>
> CoachAgent在S5的核心任务不再是"帮用户解决问题"，
> 而是"帮用户看见自己已经成为的那个人，并在那个身份上继续生长"。

---

### 0.4 知识入库的最高层筛选标准（v4.0核心新增）

v3.0的入库终极检验（"奶奶测试"）解决的是**操作化问题**——知识能不能被执行。
v4.0在此基础上增加**价值层检验**——知识是否服务于平台的最高层目标。

**四层入库筛选漏斗（必须至少通过其中一层）**：

```
第一层（痛点解决）：
  这个知识能帮助用户理解自己的健康问题吗？
  → 临床指南 / 疾病机制 / 危险因素

第二层（行为操作化）：
  这个知识能帮助用户在今天做出一个具体的行为改变吗？
  → 行为处方 / 微行为 / 环境设计

第三层（环境适应）：
  这个知识能帮助用户将新行为整合进自己的生活系统和社会关系中吗？
  → 习惯科学 / 社会支持 / 家庭关系 / 工作环境

第四层（成长超越）【v4.0新增】：
  这个知识能帮助用户发现或重新定义"我是谁"和"我的生活值得为何而改变"吗？
  → 积极心理学 / 意义框架 / 哲学思想 / 身份认同 / 自我实现
```

**新的淘汰标准**：四层均不通过的知识 = 不入库。
哪怕是权威指南，如果对平台的三层价值架构毫无贡献，也不入库（但可归档为参考文献）。

---

### 0.5 三层价值与平台功能的对应关系

| 价值层 | 阶段 | 平台功能 | CoachAgent核心任务 | 用户留下的理由 |
|--------|------|---------|------------------|--------------|
| 痛点解决 | S0-S2 | 健康评估/症状追踪/药物提醒 | 降低痛苦，提供信息 | "这里有用" |
| 痛点→适应 | S3 | 行为处方/微习惯打卡/CGM反馈 | 建立能力感和小胜利 | "我能做到" |
| 环境适应 | S4 | 习惯地图/社区支持/家庭协作 | 帮助用户巩固身份认同 | "这是我的生活方式" |
| 适应→成长 | S5 | 生命故事功能/价值观探索/成就可视化 | 帮助用户看见成为的自己 | "这里帮我看见了我自己" |
| 成长超越 | S6 | 导师角色/贡献给他人/生命叙事 | 见证用户成为他人的光 | "这里是我成长的地方" |

# 第一篇 战略与架构

## 1. 知识建设的战略定位

### 1.1 三圆飞轮与知识的关系

```
      ┌─────────────────┐
      │   底座飞轮       │  BFR/TTM/BPT6/六层管理模型
      │ 行为健康方法论    │  CrisisAgent触发条件/代谢安全红线
      └────────┬─────────┘
               │ 方法论注入
      ┌────────▼─────────┐         ┌──────────────────┐
      │   专家飞轮        │◄────────│  知识建设         │
      │ 领域知识+Agent    │         │ ★本规则的核心主题  │
      └────────┬─────────┘         └──────────────────┘
               │ 工具赋能
      ┌────────▼─────────┐
      │   教练飞轮        │  教练用Agent帮用户
      └────────┬─────────┘
               └──────────► 底座飞轮（数据反哺，闭环）
```

**根本洞见（v4.0更新）**：知识建设不是"往Agent里塞信息"，而是**把顶级专家的思维模式数字化，让教练通过AI工具获得10倍于自身的专业能力——并最终帮助用户发现和活出他们最好的版本**。

### 1.2 知识的五个层次

每一个领域的知识建设都必须覆盖五个层次，缺一则为残品：

| 层次 | 在知识包中的体现 | 典型缺失后果 |
|------|-----------------|-------------|
| **观念** | 为什么这个情境有代谢风险/成长意义？ | Agent回答"正确废话" |
| **理念** | 嵌入而非对抗；行为余量设计；成长是可能的 | 用户依从性崩塌/意义感缺失 |
| **概念** | TTM阶段/BPT6维度/M-Action优先级/价值观框架 | AI变成搜索引擎 |
| **知识** | 决策树/监测阈值/铁律边界/意义建构工具 | 边界情况出安全事故/S5用户流失 |
| **应用** | Agent话术脚本/教练行动指令/生命叙事对话框架 | 把教材交给没实习的医学生 |

### 1.3 三个"不等于"

1. **知道 ≠ 会做**：用户知道"少吃盐"不等于知道怎样少吃盐
2. **量化 ≠ 可操作**："每天钠<2000mg"对专业人员有意义，对普通用户是乱码
3. **正确 ≠ 有效**：科学正确但无法在真实场景执行的建议是无效知识
4. **【v4.0新增】有效 ≠ 有意义**：能降血糖但让用户感到"这只是为了不死"的干预，无法维持到S5-S6

### 1.4 入库的终极检验标准（v4.0双轨检验）

**轨道一（操作化检验，沿用v3.0）**：
> 一位60岁、初中文化、刚确诊高血压合并糖尿病的退休工人，看到这条建议后，能否在今天的午饭中立即开始执行——不需要查资料、不需要买工具、不需要做计算？

通不过此测试的临床/行为类知识，必须经过**第5节"傻瓜相机原则"**转化后方可入库。

**轨道二（意义层检验，v4.0新增）**：
> 这个知识，能否帮助一个已经控制好了慢性病指标的S5用户，在某一天的对话中感受到"我的改变不只是为了血糖——我是在成为一个更好的自己"？

通不过此测试的成长/哲学类知识，必须能被清晰地连接到具体的行为或身份改变，方可入库。**仅有思想深度但无落地路径的知识不入库**（可归档为参考资料）。

## 2a. 成长超越知识域（v4.0新增）

### 2a.1 成长超越域的定义

```yaml
domain: growth
domain_cn: 成长超越
定义: >
  帮助用户从"解决问题"转向"成为更好的自己"的知识体系。
  不以病症为中心，而以人的成长潜能为中心。
  包括：自我认知、生命意义、价值观澄清、身份认同、幸福科学、
  东西方哲学中的自我实现理论，以及临终关怀视角下的生命质量框架。

主要适用阶段: S4（过渡准备）/ S5（成长期）/ S6（超越期）
次要适用阶段: S2-S3（作为长期愿景锚定，降低早期放弃率）

目标: >
  帮助用户回答三个问题：
  1. 发现自我：我真正的价值观是什么？什么对我最重要？
  2. 实现自我：我想成为什么样的人？我的行为和我想成为的人是否一致？
  3. 超越自我：我如何让自己的成长对他人和世界有意义？
```

### 2a.2 成长超越域的知识来源类别

| 类别 | 具体知识方向 | 平台应用 | 优先级 |
|------|-----------|---------|-------|
| **积极心理学** | PERMA模型、心流、品格优势（VIA）、感恩科学 | S5身份重建工具 | P1 |

### 2a.3 成长超越知识的特殊证据分级

成长超越类知识的证据体系有别于临床知识，采用**双维评级**：

| 维度 | 说明 | 评级方式 |
|------|------|---------|
| **学术证据** | 有无心理学/社会学RCT或大型队列研究支持 | T1-T4沿用现有体系 |
| **文化深度** | 跨文化、跨时代的普遍性验证（如2500年的儒家实践验证） | 注明文化有效性评估（High/Medium/Low） |

> 注：积极心理学经典框架（PERMA/VIA/MBSR）有T2级学术证据；中国传统哲学框架在文化深度上为High，学术证据为T3。两者都值得入库，但检索时需分别标注，避免混淆。

### 2a.4 成长超越类知识的CoachAgent触发规则

以下情境触发成长超越域知识检索（优先于临床域）：

```yaml
触发条件（任一满足即触发）：
  - 用户表达"我不只是想控制血糖/体重，我想要更好的生活"
  - 用户在连续30天打卡后表达"然后呢？接下来呢？"
  - 用户问"你觉得什么是真正的健康？"
  - 用户提到"我好像找到了一些不一样的感觉"
  - 用户谈到家人、遗产、想留下什么
  - 用户经历重大人生事件（退休/生病/丧失/新生命）
  - BehaviorFacts显示：streak_days>90且用户开始问"为什么"类问题
  - 阶段评估判定用户进入S5

触发后CoachAgent切换模式：
  从 EXECUTION（执行指导）
  → WITNESSING（见证陪伴）
  
  核心话术方向从"怎么做"→"你已经成为了什么"
```

## 5a. 成长超越类知识的入库标准（v4.0新增）

### 5a.1 成长类知识的"落地性检验"

成长超越类知识不需要通过"奶奶测试"（即时可执行的行为操作化），
但必须通过**"生命叙事检验"**：

> **检验标准**：这个知识点，能否帮助一位用户在CoachAgent的引导下，说出一句"我以前从未这样看待自己/我的生活"的话？

通过方式：
- 提供具体的反思问题（至少3个）
- 提供可视化的框架工具（如价值观卡片、人生满意度雷达图）
- 或提供可供对话展开的叙事切入点

**不通过（禁止入库）**：
- 纯哲学论述，无任何互动工具或反思触发点
- 需要用户有高等教育背景才能理解
- 让用户感到"我做不到这么高尚"的内容（反效果）

### 5a.2 成长超越类知识的四个入库必填字段

在标准KI字段基础上，成长超越类知识额外必填：

```yaml
growth_dimension:
  # 三选一
  - discover_self      # 发现自我（价值观/身份认同/优势发现）
  - realize_self       # 实现自我（目标与行动的一致性/能力建设）
  - transcend_self     # 超越自我（贡献/传承/更大的意义）

reflection_prompts:
  # 3-5个CoachAgent可直接使用的反思性提问
  - "你说健康很重要，是因为健康让你能做什么？"
  - "如果三年后你的生活变成了你想要的样子，那会是什么样的画面？"

identity_bridge:
  # 如何将这个框架与用户的慢性病管理历程连接
  # 示例："你坚持了三个月，这说明了什么——关于你是谁这件事？"

applicable_life_events:
  # 最适合触发此知识的人生情境
  - 诊断重大疾病后
  - 退休前后
  - 孩子离家后
  - 重要人际关系改变后
```

### 5a.3 成长超越知识与痛点知识的整合原则

**禁止**：在用户还处于急性痛苦期（S0-S2早期）直接推送成长超越内容。
这会让用户感到被忽视，或感到"你在说空话"。

**正确时机**：

```
S0-S2：成长框架只能作为"远景锚定"（可能的未来），
       不作为当前干预主线。
       示例话术："很多像你这样走过来的人说，这段经历让他们
       重新思考什么对自己真正重要——当然现在咱们先把血糖稳住。"

S3-S4：成长框架作为"动力来源补充"，强化行为坚持的意义感。
       当用户问"为什么要坚持"时，超越"为了血糖"的答案。

S5-S6：成长框架成为对话主线，临床指标退为背景。
       此时CoachAgent主要功能是"见证和陪伴成长"，不是"管理疾病"。
```
