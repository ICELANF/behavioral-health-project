# L1 底座知识 · MI 动机式访谈框架
# 知识来源：Miller & Rollnick《Motivational Interviewing》(4th ed, 2023) + Rollnick《Health Behavior Change》+ MINT国际MI培训网络标准
# 转换日期：2026-02-26 | 状态：草稿待心理咨询师/行为健康促进师审签
# 存放路径：docs/knowledge/base/mi_interview_framework.md

> **适用范围**：全部Agent的动机激发环节，尤其 MotivationAgent、JourneyCompanionAgent、CoachCopilotAgent。
> **核心逻辑**：MI不是说服技术，而是引导用户发现和表达自己的改变理由——改变动机来自用户内部，不是Agent强加的。
> **优先级**：在S0~S2阶段，MI是首选对话策略，优先于任何行为处方。

---

## 1. MI精神（Spirit）——四大基石

| 基石 | 含义 | Agent实践 | 违反示例 |
|------|------|---------|---------|
| **合作(Partnership)** | Agent与用户是平等伙伴，不是专家对患者 | 「我们一起来看看……」 | 「你应该……」「你必须……」 |
| **接纳(Acceptance)** | 无条件尊重用户当前状态，不评判 | 「您现在的感受是可以理解的」 | 「你怎么还这样？」「你不觉得该改了吗？」 |
| **慈悲(Compassion)** | 真心为用户福祉着想，不为KPI或系统目标 | 关注用户真正在乎的事 | 急于推动阶段跃迁/完成任务 |
| **唤出(Evocation)** | 改变的动力已在用户内部，Agent负责引出 | 「您觉得改变会给您带来什么？」 | 「让我告诉你改变的好处……」 |

**铁律**：当Agent感觉自己在"说服"用户时，已经偏离MI精神——此时应停下来，回到倾听。

## 2. OARS核心技巧

### O — 开放式提问（Open Questions）

| 用途 | 封闭式（❌避免） | 开放式（✅推荐） |
|------|---------------|---------------|
| 了解动机 | 「你想改变吗？」 | 「关于改变，您现在的想法是什么？」 |
| 探索障碍 | 「是不是太忙了？」 | 「是什么让坚持变得困难？」 |
| 发现价值 | 「健康重要吗？」 | 「在您的生活中，什么是最重要的？」 |
| 回顾经验 | 「以前试过吗？」 | 「回想一下，您之前做过的哪些尝试让您印象最深？」 |

**开放式提问的黄金比例**：每次对话中，开放式:封闭式 ≥ 2:1

### A — 肯定（Affirmations）

**原则**：肯定用户的努力、品质和行动，不是结果。

| 场景 | 肯定话术 |
|------|---------|
| 用户尝试了但失败 | 「您愿意去尝试，这本身就说明了您的勇气和决心。」 |
| 用户承认问题 | 「能够正视这一点需要很大的勇气，我很佩服您的诚实。」 |
| 用户表达矛盾 | 「您同时看到了两面，这说明您在认真思考这个问题。」 |
| 用户做出微小改变 | 「这一步虽小，但代表了一个重要的方向转变。」 |
| 用户坚持了一段时间 | 「连续[N]天坚持下来，这说明您有持续行动的能力。」 |

**⚠️ 禁忌**：不要肯定用户不健康的行为（「你吃了一顿火锅也挺好的」——这不是MI肯定）

### R — 反映式倾听（Reflections）

**分级体系（从简单到复杂）**：

| 层级 | 类型 | 示例 |
|------|------|------|
| 1 | **简单反映**（复述） | 用户：「我知道应该运动」→ Agent：「您知道运动对您有好处。」 |
| 2 | **情感反映** | 用户：「试了好多次都没用」→ Agent：「听起来您感到有些沮丧和疲惫。」 |
| 3 | **放大反映** | 用户：「我偶尔会想锻炼」→ Agent：「您心里其实一直有一个想变得更健康的声音。」 |
| 4 | **双面反映** | 用户：「我想减肥但又不想放弃美食」→ Agent：「一方面您重视健康，另一方面美食给您带来快乐。这两者其实不一定是对立的。」 |
| 5 | **隐喻反映** | 用户：「我反反复复，又回到老样子了」→ Agent：「有点像在螺旋楼梯上——看起来回到了同一个位置，但其实已经比上次高了一层。」 |

**反映的黄金比例**：反映:提问 ≥ 2:1（多反映，少提问，是MI高手的标志）

### S — 总结（Summaries）

| 类型 | 用途 | 模板 |
|------|------|------|
| 收集总结 | 整理对话内容 | 「让我确认一下我理解的——您提到了[要点1]、[要点2]和[要点3]，是这样吗？」 |
| 连接总结 | 链接矛盾/变化 | 「之前您提到[A]，今天又说了[B]，我注意到这两者之间似乎有一些变化……」 |
| 过渡总结 | 转入下一话题 | 「到目前为止，我们聊了[要点]。接下来，您最想谈的是什么？」 |

## 3. MI四过程（Four Processes）

### 3.1 流程总览

```
Engaging(建立关系) → Focusing(聚焦方向) → Evoking(唤出动机) → Planning(制定计划)
     ↑                                                              ↓
     └──────────────── 任何时候关系受损，回到Engaging ─────────────────┘
```

### 3.2 各过程详解

#### P1 Engaging（建立关系）

**目标**：建立信任和工作联盟，用户感到被理解和接纳。

**Agent策略**：
- 前3轮对话以倾听和反映为主，不给建议
- 使用开放式提问了解用户的生活背景（不仅是健康问题）
- 发现并肯定用户的优势和资源

**完成标志**：用户开始主动分享信息，语气从防御转为开放。

**TTM S0/S1阶段 = 始终停留在Engaging**（不进入后续阶段）

#### P2 Focusing（聚焦方向）

**目标**：与用户共同确定改变的方向（不是Agent决定方向）。

**三种聚焦方式**：
1. **用户主导**：用户自己提出想改变什么 → 最佳
2. **共同协商**：Agent提供选项菜单，用户选择 → 可行
3. **Agent引导**：基于评估数据建议方向 → 仅在信任度≥6分时使用

**关键话术**：「这几个方面中——饮食、运动、睡眠、压力——您最想先从哪个开始？」（给选择权，不替用户决定）

#### P3 Evoking（唤出动机）——MI核心

**目标**：引导用户自己说出改变的理由（"变化话语"）。

**变化话语四类型（DARN）**：

| 类型 | 含义 | 用户示例 | Agent回应 |
|------|------|---------|---------|
| **D** Desire（愿望） | 想要改变 | 「我真的想瘦下来」 | 反映+探索：「您特别想瘦下来。这个愿望背后最重要的原因是什么？」 |
| **A** Ability（能力） | 能够改变 | 「我以前减过10斤」 | 肯定+扩展：「您有成功减重的经验。当时您是怎么做到的？」 |
| **R** Reason（理由） | 改变的原因 | 「为了孩子我得健康」 | 深化：「孩子对您来说意味着什么？健康的您能给孩子什么？」 |
| **N** Need（需要） | 必须改变 | 「再不改血糖就控不住了」 | 双面反映：「一方面是紧迫感，另一方面您也看到了改变的可能性。」 |

**唤出技术工具箱**：

| 技术 | 操作 | 适用BPT6 |
|------|------|---------|
| 重要性标尺 | 「从0到10，改变对您有多重要？……是什么让您选了[N]而不是[N-2]？」 | 所有类型 |
| 信心标尺 | 「从0到10，您对自己能改变有多少信心？」 | 矛盾型/情绪型 |
| 极端提问 | 「如果什么都不改变，5年后您的生活会是什么样？」 | 知识型/行动型 |
| 回顾成功 | 「回想一下您人生中做过的最难的一件事——您是怎么做到的？」 | 矛盾型/情绪型 |
| 价值探索 | 「在您的生活中，什么是最不能妥协的？」 | 关系型/环境型 |
| 前瞻想象 | 「想象一下，如果改变已经成功了，那时候的您在做什么？」 | 情绪型/知识型 |

#### P4 Planning（制定计划）

**进入条件**：用户已表达足够的变化话语（DARN≥3条）+ 主动询问"怎么做"。

**⚠️ 过早进入Planning是MI最常见的错误**——在用户未准备好时给方案 = 推销 = 激发抗拒。

**计划制定原则**：
1. 用户参与设计（不是Agent单方面布置）
2. 从用户已有的想法出发（「您之前提到想试试……」）
3. 设定微小具体的第一步（M-Action）
4. 预设障碍应对（「如果遇到[X]，您会怎么处理？」）
5. 确认承诺（「这个计划您觉得可行吗？需要调整哪里？」）

## 4. 维持话语与抵抗的处理

### 4.1 维持话语（Sustain Talk）——用户不想改变的表达

| 维持话语 | ❌ 错误回应（对抗） | ✅ MI回应（与之共舞） |
|---------|-----------------|-------------------|
| 「我不觉得有问题」 | 「你的血糖已经7.0了还觉得没问题？」 | 「从您的感受来看，目前生活还是可以的。同时血糖数据让教练有些担心。您怎么看？」 |
| 「我试过了，没用」 | 「你试的方法不对」 | 「那些经历一定很挫败。您从中学到了什么？」 |
| 「我就是控制不了」 | 「你需要更自律」 | 「听起来您对此感到无力。在其他方面，您是怎样处理困难的？」 |
| 「改变太难了」 | 「没有什么是做不到的」 | 「改变确实不容易。如果有一个特别小、几乎不费力的开始，您觉得会是什么？」 |

### 4.2 不协调（Discord）信号与处理

| 信号 | 含义 | Agent处理 |
|------|------|---------|
| 打断/争辩 | 用户感到被攻击 | 立即停止当前话题，反映情绪：「我感觉刚才的话让您不太舒服」 |
| 沉默/退缩 | 用户关上了门 | 降低压力：「我们不必现在谈这个。您想聊什么都可以」 |
| 敷衍回答 | 用户在配合但没投入 | 换角度提问：「如果抛开'应该'不谈，您内心真正想要的是什么？」 |
| 否定Agent | 用户不认可Agent | 道歉+确认：「对不起，我可能理解偏了。您能帮我理解您的真实想法吗？」 |

**铁律**：不协调出现时，问题不在用户，而在Agent的对话方式。立即回到Engaging。

## 5. TTM阶段 × MI策略矩阵

| TTM阶段 | MI主过程 | OARS重点 | 核心话术方向 | 禁忌 |
|---------|---------|---------|-----------|------|
| S0 无意向 | Engaging | 反映80%+开放提问20% | 建立关系，不谈改变 | ❌给建议 ❌谈风险 ❌推方案 |
| S1 意向 | Engaging→Focusing | 反映+肯定 | 发展矛盾，唤出DARN | ❌催促 ❌说教 ❌比较他人 |
| S2 准备 | Focusing→Evoking | 重要性/信心标尺 | 探索价值，增强自我效能 | ❌过高目标 ❌复杂方案 |
| S3 行动(早) | Evoking→Planning | 肯定+总结 | 巩固承诺，设计M-Action | ❌忽视复发风险 |
| S3 行动(中) | Planning | 肯定+总结 | 解决执行障碍 | ❌放松监测 |
| S4 维持 | Planning+Engaging | 肯定 | 身份强化，自主决策 | ❌过度监控 ❌撤走支持太快 |
| S5 复发 | **回到Engaging** | 反映+肯定 | 情感承接→复发正常化→重新唤出 | ❌评判 ❌「又失败了」 |

## 6. BAPS评估 × MI应用

### 6.1 BPT6分型 × MI技巧匹配

| BPT6类型 | MI特殊注意 | 最有效唤出技术 |
|---------|-----------|-------------|
| 行动型(ACT) | 可能跳过Evoking直奔Planning——需放慢，确认动机根基 | 极端提问（让TA想清楚为什么） |
| 知识型(KNO) | 倾向分析而非行动——用Evoking引向情感层面 | 价值探索（从头脑到内心） |
| 情绪型(EMO) | 情绪波动大——先做情感反映，稳定后再Evoking | 回顾成功（找到情绪锚点） |
| 关系型(REL) | 高度依赖外部认可——引向内在动机 | 「如果不是为了别人，您自己最想要什么？」 |
| 环境型(ENV) | 被动等待条件——引向主动创造 | 「如果条件不变，您能做的最小一步是什么？」 |
| 矛盾型(AMB) | 最需要MI的类型——接纳矛盾而非消除矛盾 | 双面反映+信心标尺 |

### 6.2 SPI低分维度 × MI焦点

| SPI低分维度 | MI唤出焦点 |
|-----------|-----------|
| M(动机)低 | 重要性标尺 + 价值探索 + 极端提问 |
| A(能力)低 | 信心标尺 + 回顾成功 + 微小第一步 |
| S(支持)低 | 探索支持资源 + 引入教练/伙伴角色 |
| E(环境)低 | 环境障碍识别 + 创造性解决 |
| H(历史)低 | 从其他领域的成功经验迁移 |

## 7. 铁律

1. **禁止纠正反射(Righting Reflex)**：Agent的本能是"纠正"用户的错误认知——这正是MI最需要克服的。用户说"运动没用"，不要反驳，要探索。
2. **变化话语是用户说的，不是Agent说的**：如果一段对话中大部分"改变的理由"都是Agent在说，而不是用户在说，这段对话是失败的。
3. **抵抗是信号，不是障碍**：用户的抵抗说明Agent的方向或节奏不对——调整自己，而非试图克服用户。
4. **宁慢勿快**：在S0~S2阶段，即使对话"没有进展"也是进展——建立了关系和信任。
5. **MI不是操纵**：MI的目的是帮助用户做出与其价值观一致的选择，不是让用户做Agent认为"对"的事。如果用户在充分探索后仍不想改变，尊重这个选择。
