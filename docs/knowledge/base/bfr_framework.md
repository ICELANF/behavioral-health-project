# L1 底座知识 · BFR 行为事实回溯六要素

> **适用范围**：全部 Agent 的「触发识别」节点均基于本框架。
> **核心逻辑**：BFR 不在于「告知」，而在于「解析」——帮助用户从受害者视角转为观察者视角。

---

## 1. BFR 定义与核心价值

BFR（Behavioral Fact Retrospection）旨在通过结构化访谈还原行为真相。

**三大价值**：
1. 从「情绪脑」向「理性脑」的强制迁移——描述具体细节时，杏仁核负荷降低
2. 识别「功能性需求」而非「道德缺陷」——几乎所有不良行为都在解决真实问题
3. 建立 0.1 秒觉察窗口——在未来触发瞬间多一点暂停空间

---

## 2. BFR 六要素

| 要素 | 含义 | Agent 询问示例 |
| :--- | :--- | :--- |
| **B (Background)** | **背景/诱因** | "在你想喝奶茶的前一刻，发生了什么？" |
| **F (Food/Action)** | **具体行为** | "这顿火锅你具体吃了什么？" |
| **R (Reaction)** | **身心反应** | "吃完之后，你的身体和心情感觉如何？" |
| **S (Space)** | **物理空间** | "当时你在哪里？" |
| **T (Time)** | **时间节点** | "这种渴望通常发生在下午几点？" |
| **P (People)** | **社交情境** | "当时身边有谁？" |

---

## 3. 应用规范

* **强制回溯原则**：给出 M-Action 前，必须至少完成 B 和 F 的回溯。
* **事实大于结论**：通过具体细节询问验证用户的概括性描述（「我总是」→「上次是什么时候？」）。
* **禁止反向追问**：严禁在用户填写 BFR 时说「你当时为什么不……」，这会激活防御反应。

---

## 4. 无效回溯 vs 有效回溯对比

| 要素 | 无效示例（用户情绪化描述） | 有效示例（BFR 逻辑还原） |
| :--- | :--- | :--- |
| 描述 | "我昨晚太没出息了，又吃了火锅。" | "昨晚 20:15，在单位加班时，点了一份火锅。" |
| 触发 | "我就是想吃辣的。" | "主管在 20:00 驳回了我的方案，感到胸口发闷。" |
| 反应 | "觉得我这辈子减不了肥了。" | "念头：'我都这么累了，吃点热乎的怎么了？'" |
| 动作 | "疯吃一顿，停不下来。" | "5分钟内下单，外卖到后，边看手机短视频边吃。" |
| 结果 | "恨死自己了。" | "满头大汗时感到压力释放；30分钟后心跳加快，反酸。" |

> **核心提示语**：「BFR 不是为了让我们看清过去，而是为了让我们在未来的那个'触发瞬间'，拥有多 0.1 秒的觉察力。这 0.1 秒，就是自由的开始。」
