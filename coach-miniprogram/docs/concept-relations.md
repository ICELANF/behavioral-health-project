# 前端 · 后端 · UI · 路由 · API · 端口 — 逻辑关系导图

## 1. 全局关系图

```
用户 (手指点屏幕)
 │
 │  看到的、操作的
 ▼
┌─────────────────────────────────────────────────────┐
│                 用户界面 (UI)                         │

│                                                      │
│  技术实现: Vue 模板 (<template> 里的 HTML/CSS)        │
└──────────────────────┬──────────────────────────────┘

                       ▼

│                                                      │

│  │  (.vue模板)  │  │  (pages.json) │  │ (request.ts)│  │

│  └──────┬──────┘  └──────┬───────┘  └─────┬──────┘  │
│         │               │                │          │
│  点击按钮 ──→ 切换页面路由 ──→ 发送 HTTP 请求 ──→    │

                                              │
                    网络传输 (HTTP请求)         │
                    经过端口                    │
                                              ▼
┌─────────────────────────────────────────────────────┐
│                    端口 (Port)                        │
│  "大楼的门牌号"                                       │

                       │ 请求到达 :8000

│     ◄── 返回 JSON ◄──── 组装结果 ◄──── 查询结果      │
```

## 2. 核心概念定义 + 类比

```
┌──────────┬──────────────────────────┬──────────────────────────┐
│  概念     │  是什么                   │  类比                     │
├──────────┼──────────────────────────┼──────────────────────────┤
│  UI      │  用户看到和操作的界面      │  餐厅的菜单和装修          │
│  前端     │  在用户设备上跑的代码      │  服务员 (接单、传菜)       │
│  后端     │  在服务器上跑的代码        │  厨房 (做菜、管库存)       │
│  路由     │  URL 路径 → 对应处理      │  地址导航 (XX路→XX店)     │
│  API     │  前后端之间的通信契约       │  点菜单 (菜名+价格+规格)  │
│  端口     │  同一机器上区分服务的编号   │  大楼的房间号              │
└──────────┴──────────────────────────┴──────────────────────────┘
```

## 3. "路由" 存在于两个地方

```
               路由 (Route)
            "路径 → 处理者 的映射"
               ┌────┴────┐
               │         │
        ┌──────▼──────┐  ┌──────▼──────┐
        │  前端路由    │  │  后端路由    │
        │             │  │             │
        │ URL → 页面  │  │ URL → 函数  │
        │             │  │             │
        │ /pages/     │  │ /api/v1/    │
        │ coach/      │  │ coach/      │
        │ dashboard   │  │ dashboard   │
        │   ↓         │  │   ↓         │
        │ 显示仪表盘  │  │ 返回JSON    │
        │ 页面组件    │  │ 数据        │
        └─────────────┘  └─────────────┘

  pages.json 定义        FastAPI @router.get()
  "用户访问哪个页面"      "前端请求打到哪个函数"
```

## 4. "API" 是前后端的契约

```
        前端                    API 契约                    后端
  ┌──────────────┐     ┌───────────────────┐     ┌──────────────┐
  │              │     │                   │     │              │
  │  http.get(   │     │  GET /v1/coach/   │     │  @router.get │
  │  '/v1/coach/ │ ──→ │    dashboard      │ ──→ │  ("/dashboard│
  │   dashboard')│     │                   │     │  ")          │
  │              │     │  请求: Header带JWT │     │  async def   │
  │  期望返回:    │     │  响应: {          │     │  get_dash(): │
  │  {coach,     │     │    coach: {...},  │     │    return {} │
  │   today_stats│     │    today_stats:{} │     │              │
  │   students}  │     │    students: []   │     │              │
  │              │     │  }                │     │              │
  └──────────────┘     └───────────────────┘     └──────────────┘

  API = 双方约定的:
    ① 路径 (URL)
    ② 方法 (GET/POST/PUT/DELETE)
    ③ 请求格式 (body/params)
    ④ 响应格式 (JSON 结构)
```

## 5. 一次完整请求的生命周期

```
 ❶ 用户点击 "学员列表"
    │
 ❷ UI 触发事件 → @tap="loadStudents()"
    │
 ❸ 前端路由跳转 → uni.navigateTo('/pages/coach/students/index')
    │
 ❹ 页面 onMounted → http.get('/v1/coach/students')
    │
 ❺ HTTP 模块拼接完整 URL
    │  http://localhost:8000/api/v1/coach/students
    │                ─┬──── ─┬─ ──────┬──────────
    │                 │      │        │
    │              域名+端口  │    API 路径
    │                     前缀
    │
 ❻ 请求经网络 → 到达后端 :8000
    │
 ❼ 后端路由匹配 → /v1/coach/students → get_students()
    │
 ❽ 业务逻辑 → 查数据库 → 组装 JSON
    │
 ❾ 返回 { items: [...], total: 4 }
    │
 ❿ 前端收到 JSON → 赋值给 ref → Vue 响应式更新 UI
    │
    ▼
 用户看到 4 个学员卡片
```

## 7. 逻辑包含关系

```
前端 ─┬─ UI (用户界面)
      │    └─ 用户看到的所有视觉元素
      ├─ 前端路由
      │    └─ 页面之间的导航规则
      └─ HTTP 模块
           └─ 调用 API 的工具

后端 ─┬─ 后端路由
      │    └─ URL → 处理函数 的映射
      ├─ API 处理 (业务逻辑)
      │    └─ 验证、计算、调用 Agent
      └─ 数据层
           └─ 数据库、缓存、向量库

端口 ── 不属于前端也不属于后端
         它是操作系统层的概念
         让同一台机器上的多个服务各占一个"门"

API ─── 不属于前端也不属于后端
         它是双方之间的"契约"
         前端按契约发请求，后端按契约返数据
```
