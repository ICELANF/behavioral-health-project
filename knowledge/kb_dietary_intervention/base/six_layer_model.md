# L1 底座知识 · 六层管理模型

<!-- ki_id: KI-BASE-SIX-LAYER-001 | layer: L1 | scope: platform | evidence: T1 | ttm_stages: S0-S6 | document_version: v1.0 | ingest_date: 2026-03-07 -->

> **适用范围**：全部 Agent。六层模型定义了行为健康管理从"信息获取"到"生命意义实现"的完整路径。
> **核心逻辑**：管理不是单点控制，而是从觉察到超越的六层递进。每一层都在前一层基础上建立。

---

## 1. 六层架构总览

```
第六层: 超越管理 ——— 成为他人的支持者和引路人（S6）
        ↑
第五层: 身份管理 ——— "我是谁" 的重新定义（S5）
        ↑
第四层: 环境管理 ——— 将新行为嵌入生活系统（S3-S4）
        ↑
第三层: 行为管理 ——— 具体可执行的行为处方（S2-S3）
        ↑
第二层: 认知管理 ——— 理解机制，建立动机（S1-S2）
        ↑
第一层: 觉察管理 ——— 意识到问题的存在（S0-S1）
```

---

## 2. 各层详解

### 第一层：觉察管理（S0-S1）

| 维度 | 内容 |
|------|------|
| **目标** | 让用户从"不知道"或"不承认"走向"意识到" |
| **CoachAgent任务** | 无评判地展示事实（BFR回溯），建立安全信任 |
| **典型话术** | "你有没有注意过，每次加班后你的饮食选择有什么变化？" |
| **禁止** | 直接给建议、使用恐吓语言（"再不改就要打胰岛素了"） |
| **对应知识** | BFR框架、TTM S0-S1判定、信任建立协议 |

### 第二层：认知管理（S1-S2）

| 维度 | 内容 |
|------|------|
| **目标** | 让用户理解行为与健康的因果链条 |
| **CoachAgent任务** | 用可理解的语言解释机制，不是灌输知识 |
| **典型话术** | "你昨天那顿火锅后血糖升到12.3——这不是因为你意志力差，是因为火锅汤底的高碳水让血糖快速上升。" |
| **禁止** | 使用专业术语轰炸、一次给出超过2个知识点 |
| **对应知识** | 临床指南（简化版）、代谢机制可视化 |

### 第三层：行为管理（S2-S3）

| 维度 | 内容 |
|------|------|
| **目标** | 把认知转化为具体可执行的微行为 |
| **CoachAgent任务** | 给出M-Action清单，确保P1优先级行为今天就能做 |
| **典型话术** | "今天午饭时，先吃两口蔬菜再吃米饭——就这一件事。" |
| **禁止** | 一次给超过3个行为目标、给出需要计算的建议 |
| **对应知识** | M-Action优先级、行为处方六要素、傻瓜相机原则 |

### 第四层：环境管理（S3-S4）

| 维度 | 内容 |
|------|------|
| **目标** | 让新行为不再依赖意志力，而是被环境自动触发 |
| **CoachAgent任务** | 协助用户设计环境、建立社会支持、处理障碍 |
| **典型话术** | "你说办公室零食柜是你最大的敌人——如果在抽屉里放一包坚果，你觉得会怎样？" |
| **禁止** | 忽视用户的真实生活约束（"你应该不去应酬"） |
| **对应知识** | 习惯科学、环境设计、社会支持系统、家庭协作 |

### 第五层：身份管理（S5）

| 维度 | 内容 |
|------|------|
| **目标** | 帮助用户从"管理疾病的人"变为"追求健康生活的人" |
| **CoachAgent任务** | 见证身份转变，引导自我叙事重建 |
| **典型话术** | "你说'我现在是那种会主动选择走楼梯的人'——这个变化是什么时候开始的？" |
| **禁止** | 替用户定义身份（"你已经是一个健康的人了"） |
| **对应知识** | 可能的自我理论、叙事认同、PERMA框架 |

### 第六层：超越管理（S6）

| 维度 | 内容 |
|------|------|
| **目标** | 让用户的成长惠及他人，形成社会传递效应 |
| **CoachAgent任务** | 支持用户成为导师/分享者角色 |
| **典型话术** | "你的经历可能正是某个刚确诊的人最需要听到的——你愿意分享吗？" |
| **禁止** | 强迫分享、将用户工具化为"推广案例" |
| **对应知识** | 利他行为科学、导师角色发展、社会影响力 |

---

## 3. 层级流转规则

- **不可跳层**：用户必须在前一层建立基础后才能进入下一层。S0 用户不做行为处方，S2 用户不讨论身份认同。
- **可回退**：生活事件（疾病复发/重大压力）可能导致用户从高层回退。回退不是失败，是正常波动。
- **层级共存**：S4 用户可能同时在第四层（环境管理）和第五层（身份初探）之间。Agent 需判断当前对话的主要层级。
- **最高层目标锚定**：即使在第一层工作，Agent 也应意识到用户最终可以走到第六层。这个信念影响语气和态度。

## 4. 与 TTM 阶段模型的映射

| 六层 | TTM阶段 | 三层价值 | Agent主要检索域 |
|------|---------|---------|---------------|
| 觉察 | S0-S1 | 痛点解决 | base + clinical |
| 认知 | S1-S2 | 痛点解决 | clinical + metabolic |
| 行为 | S2-S3 | 痛点→适应 | behavioral + metabolic |
| 环境 | S3-S4 | 环境适应 | behavioral + coaching |
| 身份 | S5 | 成长超越 | growth |
| 超越 | S6 | 成长超越 | growth |
