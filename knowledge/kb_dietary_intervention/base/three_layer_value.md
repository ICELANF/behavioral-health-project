# L1 底座知识 · 三层价值架构（痛点→适应→成长超越）

<!-- ki_id: KI-BASE-THREE-LAYER-001 | layer: L1 | scope: platform | evidence: T1 | ttm_stages: S0-S6 | document_version: v1.0 | ingest_date: 2026-03-07 -->

> **适用范围**：全部 Agent 的阶段识别与模式切换逻辑。CoachAgent 在 S4→S5 转折点的核心决策依据。
> **核心逻辑**：慢性病管理是入口，不是目的。行为改变是路径，不是终点。最高目标是人的成长与生命意义的实现。

---

## 1. 三层价值定义

| 价值层 | TTM阶段 | 用户核心问题 | CoachAgent模式 | 用户留下的理由 |
|--------|---------|-------------|---------------|--------------|
| **痛点解决** | S0-S2 | "我的血糖怎么降？" | EXECUTION（执行指导） | "这里有用" |
| **环境适应** | S3-S4 | "怎么把新习惯融入生活？" | COACHING（能力建设） | "我能做到" |
| **成长超越** | S5-S6 | "我想成为什么样的人？" | WITNESSING（见证陪伴） | "这里帮我看见了自己" |

---

## 2. 各层不可缺失性

- **痛点层缺失** → 没有入口，平台没有用户
- **适应层缺失** → 痛点缓解后用户流失，行为无法持续
- **成长层缺失** → S4→S5 的玻璃天花板，平台永远只是健康管理App

---

## 3. CoachAgent 阶段切换决策树

```
用户当前阶段判定
├── S0-S2（痛点期）
│   ├── Agent模式: EXECUTION
│   ├── 知识检索: filter domain=metabolic/clinical, layer=L1/L2
│   ├── 话术方向: "怎么做" — 具体可操作的行为指令
│   └── 成长框架: 仅作远景锚定，不作主线
│       └── 示例: "很多走过来的人说，这段经历让他们重新思考什么对自己真正重要——当然现在先把血糖稳住。"
│
├── S3-S4（适应期）
│   ├── Agent模式: COACHING
│   ├── 知识检索: filter domain=behavioral/metabolic, layer=L2/L3
│   ├── 话术方向: "你已经能做到了" — 强化自我效能
│   └── 成长框架: 作为动力来源补充
│       └── 示例: "你坚持了三个月，这不只是为了血糖——你觉得这说明你是一个什么样的人？"
│
└── S5-S6（成长期）
    ├── Agent模式: WITNESSING
    ├── 知识检索: filter domain=growth, ttm_stages contains S5
    ├── 话术方向: "你已经成为了什么" — 见证身份成长
    └── 临床指标: 退为背景监测，不作对话主线
        └── 示例: "你刚才说的——'我发现我真的在乎活得有质量'——这是什么时候开始的？"
```

## 4. S4→S5 关键转折点识别

**转折的本质**：用户从"我有什么问题"开始转向"我是谁"。

### 4.1 识别信号

| 信号类型 | 具体表现 | 触发动作 |
|---------|---------|---------|
| 语义信号 | "我不只是想控制血糖，我想要更好的生活" | 启动成长域知识检索 |
| 行为信号 | streak_days > 90 且用户开始问"为什么"类问题 | 建议阶段评估 |
| 情境信号 | 用户提到退休/重大疾病/家人/遗产/想留下什么 | 切换WITNESSING模式 |
| 元认知信号 | "然后呢？接下来呢？""坚持下去的意义是什么？" | 引入意义探索对话 |

### 4.2 转折引导话术（5个）

1. "你坚持了这么久，我很好奇——是什么让你一直没有放弃？"
2. "你说'不只是为了血糖'——那你觉得，更深层的原因是什么？"
3. "如果三年后你的生活变成了你真正想要的样子，那会是什么画面？"
4. "很多人在你这个阶段，开始发现自己变了——不只是身体，是整个人。你有没有类似的感觉？"
5. "你现在的生活方式和一年前相比，最大的不同是什么？这个不同对你意味着什么？"

### 4.3 自我叙事对比

```
S4叙事: "我是一个努力控制糖尿病的人。"
  → 身份仍然围绕疾病定义
  → CoachAgent任务: 巩固习惯，预防倒退

S5叙事: "我是一个重视健康、热爱生活的人，糖尿病是我人生故事的一部分。"
  → 身份从疾病中解放出来
  → CoachAgent任务: 见证成长，帮用户看见自己已经成为的那个人
```

## 5. 铁律

- **铁律7（成长框架时机）**：S0-S2 前期禁止以成长框架作为干预主线。违反后果：用户感到"被看穿但不被接住"，直接流失。
- **铁律8（最高目标）**：所有知识入库决策必须对照三层价值架构中至少一层有贡献。不服务于任何一层的知识不入主库。
