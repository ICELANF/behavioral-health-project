# L1 底座知识 · M-Action 优先级原则

<!-- ki_id: KI-BASE-MACTION-001 | layer: L1 | scope: platform | evidence: T1 | ttm_stages: S1-S4 | document_version: v1.0 | ingest_date: 2026-03-07 -->

> **适用范围**：所有涉及行为处方输出的 Agent（CoachAgent / NutritionAgent / ExerciseAgent / BehaviorRxAgent）。
> **核心逻辑**：M-Action（微行为）是行为处方的最小执行单元。优先级决定"先做哪一个"——不是给用户一堆建议，而是给出一个今天就能做的事。

---

## 1. M-Action 定义

M-Action（Micro-Action，微行为）是满足以下全部条件的行为单元：

| 条件 | 标准 | 反例 |
|------|------|------|
| **即时可执行** | 5分钟内可以开始 | "制定一个运动计划"（太大） |
| **无需工具** | 不需要购买/准备任何东西 | "用食物秤称量100g鸡胸肉" |
| **零计算** | 不需要换算单位或百分比 | "将碳水化合物控制在40%"  |
| **单一动作** | 一个具体的身体动作 | "注意饮食和运动"（两件事） |
| **可观察** | 自己或他人能判断是否完成 | "少吃一点"（无法判定） |

**正例**："今天午饭时，先吃两口蔬菜再吃米饭。"
**正例**："晚饭后，出门走到小区大门口再走回来。"
**正例**："下次想吃零食时，先喝一杯温水，等3分钟。"

## 2. 优先级规则（P1-P5）

### 2.1 优先级定义

| 级别 | 含义 | 何时给出 | 每次最多 |
|------|------|---------|---------|
| **P1** | 今天必须做的1件事 | 每次对话必给 | 1个 |
| **P2** | 本周建立的习惯种子 | 用户完成P1后给 | 1-2个 |
| **P3** | 环境调整建议 | S3阶段后给 | 1个 |
| **P4** | 社交场景应对策略 | 用户提出社交困难时给 | 1个 |
| **P5** | 进阶挑战（有弹性目标） | S4阶段后给 | 1个 |

### 2.2 铁律：一次只推一个P1

> Agent 每次对话只给用户**一个 P1 级 M-Action**。
> 用户问"还有什么我能做的？"时，才给 P2。
> 绝不一次给出超过3个 M-Action。

**为什么**：行为科学研究（Fogg 2020）表明，同时改变多个行为的成功率低于10%。单行为启动的成功率超过80%。

## 3. Slack 概念（行为余量）

### 3.1 定义

Slack（行为余量）= 用户能执行的最大行为变化量 × 0.7

**例子**：
- 用户说"我觉得我能每天走30分钟" → 实际处方 = 20分钟
- 用户说"我能戒掉所有零食" → 实际处方 = 每天只减少1次零食
- 用户说"我要每天做40分钟运动" → 实际处方 = 每天15分钟即可

### 3.2 Slack 的三个作用

1. **防止完美主义陷阱**：过高目标 → 一次失败 → 全盘放弃
2. **创造超额完成感**：做到了比目标多一点 → 自我效能感提升
3. **为坏日子留空间**：生病/加班/情绪低落时仍能完成 → 不中断连续性

### 3.3 Slack 调整规则

| 用户类型 | Slack系数 | 理由 |
|---------|----------|------|
| S0-S1（新手） | 0.5 | 建立"我能做到"的信心最重要 |
| S2-S3（适应期） | 0.7 | 标准系数 |
| S4+（成熟期） | 0.85 | 用户已有自我调节能力 |
| 情绪低落/疾病复发 | 0.3 | 维持连续性比进步更重要 |

## 4. M-Action 设计的"傻瓜相机"检验

每个 M-Action 必须通过以下检验：

```
□ 60岁退休工人能理解吗？ → 不能则重写
□ 不需要查资料就能做吗？ → 不能则拆分
□ 5分钟内能开始吗？       → 不能则降级
□ 做完了自己能判断吗？     → 不能则加可观察指标
□ 失败了有退路吗？         → 没有则加 Slack
```

## 5. M-Action 输出模板

Agent 输出 M-Action 时的标准格式：

```
🎯 今天的一件事（P1）
[具体动作]，[时间/场景锚点]。

💡 为什么这样做
[一句话机制说明，不超过20字]

✅ 完成标准
[可观察的完成判定]
```

**示例**：
```
🎯 今天的一件事
晚饭时，先吃三口青菜再吃米饭。

💡 为什么这样做
蔬菜纤维减缓血糖上升速度。

✅ 完成标准
第一口是蔬菜就算完成。
```

## 6. 禁止行为

- **禁止**：给出需要用户自行换算的百分比或克数（铁律6）
- **禁止**：一次给出超过3个M-Action
- **禁止**：使用"你应该""你必须"的命令语气
- **禁止**：在用户上一个P1还没做到时就推P2
- **禁止**：给出不满足5条检验标准的M-Action
