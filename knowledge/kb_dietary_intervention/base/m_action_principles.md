# 行健平台M行动原则 | BHP M-Action Principles

> 文档版本：v4.0 | 创建：2026-03-07 | 遵循BHP知识库建设规则v4.0 | 行健平台知识库
> **M = Minimum（最小）× Meaningful（有意义）× Maintainable（可维持）**

---

## 概述

M行动原则是行健平台行为处方生成的**核心设计哲学**。所有AI代理、教练顾问在生成行为建议时，必须以M原则为第一验证标准。M原则来源于行为科学研究的核心发现：**行为改变最大的障碍不是动机不足，而是行为门槛过高。**

```
M行动三角:
         ▲
        /M\
       / i \
      / n i \
     /  n m \
    / i  a u \
   /  m  l m \
  / u  u  e \
 / m  m  d \
/____________\
Minimum  Meaningful  Maintainable
（最小）  （有意义）  （可维持）
```

---

## 原则一：Minimum（最小可执行）

### 核心命题
**找到最小可行行为（Minimum Viable Behavior, MVB）**——小到用户在毫无动力的最坏情况下也能做到。

### 设计规则

```yaml
MVB判断标准:
  - 执行时间: ≤ 2分钟（入门阶段）
  - 所需意志力: 极低（不需要"振作"就能做）
  - 所需技能: 已具备（不需要先学习）
  - 所需资源: 触手可及（不需要提前准备）

失败案例（过高门槛）:
  ❌ "每天跑步30分钟"
  ❌ "完全戒除精制碳水"
  ❌ "每天记录三餐热量"

MVB改造版:
  ✅ "穿上运动鞋走出门" （跑步的最小版本）
  ✅ "每顿饭先吃一口蔬菜" （饮食改变的最小版本）
  ✅ "拍一张今天午餐照片" （记录的最小版本）
```

### 傻瓜相机检验法
> 在生成任何行为处方之前，问自己：
> **"一个62岁、膝盖不好、刚确诊糖尿病的退休男性，今天第一次看到这个建议，他能立刻做到吗？"**
> 若答案是"否"或"不确定"，继续缩小行为粒度。

### 进阶设计：行为宽容区间
```
最大版本（高动力时）   ←→   最小版本（低动力时）
"跑步30分钟"               "出门走5分钟"
"完整健康餐"               "饭前喝一杯水"
"正念冥想20分钟"           "深呼吸3次"
```
同一用户同一习惯，根据当天状态在区间内选择，避免"全或无"失败。

## 原则二：Meaningful（有意义）

### 核心命题
**行为必须与用户的核心价值观和重要目标相连接**。没有意义感的行为是纯粹的意志力消耗，注定不可持续。

### 意义层次模型

```
Level 3: 超越意义（为他人/社会）
  "我保持健康，这样能陪孩子更长时间"
  "我成为了家人的榜样"
        ↑ 最持久
Level 2: 身份意义（我是谁）
  "作为一个自律的人，这是我应该做的"
  "这符合我对自己的期待"
        ↑ 持久
Level 1: 工具意义（达成目标）
  "我需要降低血糖"
  "我想减掉10斤"
        ↑ 有效但易消退
```

### 意义探索技术（5 Whys）
```
用户说："我想减肥"
↓ Why 1: 为什么想减肥？
"医生说我血糖高了"
↓ Why 2: 为什么血糖高对你重要？
"我怕糖尿病并发症"
↓ Why 3: 为什么你担心并发症？
"我不想像我父亲一样晚年失去行动能力"
↓ Why 4: 为什么行动能力对你重要？
"我想在孙子长大的时候还能陪他们玩"
↓ Why 5: 这对你意味着什么？
"我的生命意义之一是见证下一代成长"
→ 真正的驱动力找到了：祖父/祖母角色认同
```

### 意义连接话术
> "您做这件事，最终是为了什么？"
> "当您想象三年后健康改变后的生活，您看到了什么？"
> "在您最在乎的人眼中，这个改变意味着什么？"

### 意义提醒设计
- 个性化推送：将用户的意义陈述嵌入每日提醒文案
- 进度与意义绑定："您已经坚持30天，这意味着您正在成为您想成为的那个人"
- 里程碑庆祝：强调里程碑的意义维度，不只是数字

## 原则三：Maintainable（可维持）

### 核心命题
**行为必须能在用户生活的真实约束下长期执行**——出差时、节假日时、生病时、高压时。不可维持的行为是定时失效的假改变。

### 可维持性设计五要素

**1. 场景鲁棒性**
```
为每个目标行为设计至少3个场景版本：
  标准版：正常日子执行
  旅行版：出差/旅游时的替代方案
  压力版：极忙/焦虑时的最小版本
```

**2. 失误恢复计划**
```
"如果我___（预期失误），我会___（恢复动作）"

示例：
  "如果我连续3天没有运动，我会第4天穿鞋出门走5分钟，不追加补偿"
  "如果聚餐吃多了，我不会用节食惩罚自己，下一顿正常吃"

核心原则：失误≠失败，复原速度比完美执行更重要
```

**3. 社会情境适应**
```
识别用户生活中的"破坏性情境"：
  - 家庭聚餐（饮食压力）
  - 同事应酬（饮酒压力）
  - 节假日（日程打乱）
  - 家庭压力事件（情绪性进食触发）

预先设计应对脚本（if-then计划）
```

**4. 渐进增量设计**
```
不要一次部署所有改变。行为堆叠原则：
  第1-2周: 只建立一个习惯锚点
  第3-4周: 在稳定基础上添加第二个
  每月: 评估并调整
```

**5. 成本-收益动态平衡**
```
当收益>成本：行为自然持续
当成本>收益：行为被放弃

教练/AI任务：
  → 降低成本（减少摩擦/提供支持）
  → 增加感知收益（可视化数据/即时反馈）
  → 帮助用户看到长期收益
```

## M原则的整合应用

### 行为处方质量审核清单

在生成任何行为建议前，逐项检验：

```
M1 - Minimum:
  □ 这个行为最坏情况下能做到吗？（傻瓜相机测试）
  □ 有没有更小的版本？
  □ 是否设计了最小版本/最大版本区间？

M2 - Meaningful:
  □ 这个行为连接了用户的核心价值观吗？
  □ 用户能看到这个行为与其生命意义的关系吗？
  □ 有没有做过5 Whys探索？

M3 - Maintainable:
  □ 出差/节假日/生病时有替代方案吗？
  □ 有没有失误恢复计划？
  □ 是否符合渐进增量原则（没有一次要求太多）？
```

### 常见失败模式与M原则对照

| 失败模式 | 违反原则 | M原则修正 |
|---------|---------|---------|
| 开始热情，三天放弃 | M1（门槛太高） | 找到最小版本 |
| 知道重要但不想做 | M2（意义未连接） | 5 Whys探索真正动机 |
| 旅行后再也没回来 | M3（无场景鲁棒性） | 设计旅行版行为 |
| 失误一次全部放弃 | M3（无失误恢复计划） | "失误后的下一步"预设 |
| 一次改太多全失败 | M3（违反渐进原则） | 一次只建一个习惯 |
