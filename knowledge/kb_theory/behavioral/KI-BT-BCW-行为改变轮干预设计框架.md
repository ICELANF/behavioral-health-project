## KI-BT-BCW-001：BCW整体框架——三层同心圆结构

```yaml
id: KI-BT-BCW-001
type: 行为理论
evidence: T2
stage: S0-S5
domain: behavioral
keywords: [行为改变轮, BCW, COM-B, 干预函数, 政策类别, 三层框架]
```

**行为改变轮（BCW）的核心结构**

BCW是对19个现有行为改变框架进行系统综合的结果，由三层同心圆组成：

- **最内层（轮毂）：COM-B模型**——描述行为发生的心理机制（能力×机会×动机=行为）
- **中间层：9大干预函数**——基于COM-B诊断确定干预方向（教育、说服、激励、强制、训练、限制、环境重塑、示范、赋能）
- **外层：7类政策类别**——决定干预如何落地实施（传播营销、指导方针、财政措施、监管、立法、环境/社会规划、提供服务）

**BCW的核心价值**：提供系统方法，确保干预设计者全面考虑所有可能选项，而非基于惯性或直觉选择干预方式。BCW不是"行为改变魔法弹"，而是将现有理解和资源最优化利用的决策系统。

**平台应用**：CoachAgent在为用户设计行为处方时，应基于COM-B诊断→选择干预函数→匹配BCT三步走，而非直接给出建议。

**Agent决策逻辑**：当用户新启动某个行为目标时，先问"是能力问题（不会做）、机会问题（没条件做）还是动机问题（不想做）"，再选干预方向。

## KI-BT-BCW-002：COM-B模型——行为分析六维框架

```yaml
id: KI-BT-BCW-002
type: 行为理论
evidence: T2
stage: S0-S5
domain: behavioral
keywords: [COM-B, 能力, 动机, 机会, 行为诊断, 反思性动机, 自动性动机]
```

**COM-B六维定义**

| COM-B维度 | 定义 | 典型案例 |
|-----------|------|---------|
| **身体能力** | 执行行为所需的体力、技能、耐力 | 能否完成深蹲动作 |
| **心理能力** | 执行行为所需的知识、心理技能、心理耐力 | 知道如何计算碳水克数 |
| **身体机会** | 物理环境提供的条件：时间、资源、地点、提示 | 家附近有健身房且买得起运动鞋 |
| **社会机会** | 人际影响提供的条件：社会规范、文化期待、他人支持 | 家人一起吃饭时支持低油饮食 |
| **反思性动机** | 有意识的计划与评估（相信什么好/坏，想要什么） | 认为减重对健康有益并制订计划 |
| **自动性动机** | 无意识的欲望、冲动、习惯、反射 | 看到薯片自动想吃（旧习惯）|

**核心原理**：行为改变失败的根本原因往往是"以为是动机问题，实则是能力或机会问题"（反之亦然）。精准诊断COM-B维度是有效干预的前提。

**行为操作化（傻瓜相机原则）**：CoachAgent在会话中，用三个问题快速诊断：
1. "您知道怎么做吗？"→ 能力
2. "您有时间/工具/环境支持吗？"→ 机会
3. "您有没有特别想做的感觉？"→ 动机

**监测指标**：定期用COM-B自评问卷复查各维度，动态调整干预策略。

## KI-BT-BCW-003：COM-B行为规范化描述——WHO-WHAT-WHERE-WHEN四维法

```yaml
id: KI-BT-BCW-003
type: 行为操作工具
evidence: T2
stage: S1-S4
domain: behavioral
keywords: [行为规范化, 行为诊断, 目标行为设定, 具体化, 可测量]
```

**为什么必须规范化描述行为**

模糊行为目标→干预无的放矢。"多运动"对干预设计毫无用处；"每周三和周六上午9点在小区内快走60分钟"才能指导行动。NICE的研究表明，把行为目标从笼统描述改为规范化描述，显著提高了干预有效性（英国NICE随后改变了指南写作规范）。

**四维行为规范化模板**

| 维度 | 问题 | 错误示范 | 正确示范 |
|------|------|---------|---------|
| **Who** | 谁需要执行此行为？ | "患者" | "2型糖尿病患者中年龄>50岁的工作者" |
| **What** | 具体做什么（与当前有何不同）？ | "多运动" | "每次餐后10分钟在室内或室外步行" |
| **Where** | 在哪里执行？ | "户外" | "饭后在小区花园或居家客厅" |
| **When** | 何时、多久一次？ | "经常" | "三餐后各10分钟，每天3次" |

**CoachAgent话术应用**：当用户说"我想好好吃饭"时，引导规范化：
- "好，我们来让这个目标更清晰——您想在哪顿饭上先做改变？早饭、午饭还是晚饭？"
- "这顿饭您通常在哪里吃？家里还是单位食堂？"
- "您希望具体改变什么——主食量、蔬菜量、吃的顺序，还是其他？"

**铁律**：不要接受模糊目标直接进入行动计划。必须完成Who-What-Where-When四维规范化才算目标确认。

## KI-BT-BCW-004：9大干预函数——定义、示例与COM-B映射

```yaml
id: KI-BT-BCW-004
type: 干预策略
evidence: T2
stage: S0-S5
domain: behavioral
keywords: [干预函数, 教育, 说服, 激励, 训练, 环境重塑, 赋能, 示范, 限制, 强制]
```

**BCW 9大干预函数完整定义**

| 干预函数 | 定义 | 典型应用场景 | 靶向COM-B |
|---------|------|------------|-----------|
| **教育** | 增加知识或理解 | 向用户解释高血糖对血管的长期损伤 | 心理能力 |
| **说服** | 通过沟通激发积极/消极情绪或促进行动 | 分享同类用户成功案例视频激励行动 | 反思性动机 |
| **激励** | 创造对奖励的期待 | 设置打卡积分兑换奖励 | 反思性/自动性动机 |
| **强制** | 创造对惩罚或代价的期待 | 公开承诺给朋友监督 | 反思性动机 |
| **训练** | 传授技能 | 教用户如何正确测餐后血糖 | 身体/心理能力 |
| **限制** | 通过环境减少机会（以减少不良行为） | 不在家中存放高糖零食 | 身体机会 |
| **环境重塑** | 改变物理或社会环境 | 把运动鞋放在门口作为视觉提示 | 身体/社会机会 |
| **示范** | 提供他人榜样供效仿 | 展示同类群体的健康生活方式 | 反思性动机 |
| **赋能** | 超越教育训练，增加能力或减少行为障碍 | 提供CGM设备、无障碍健身资源转介 | 能力+机会综合 |

**平台CoachAgent应用决策树**：
- 用户不知道怎么做 → 教育 + 训练
- 用户知道但不想做 → 说服 + 激励
- 用户想做但没条件 → 环境重塑 + 赋能
- 用户需要长期维持 → 示范（社群） + 激励（积分）
- 用户有不良习惯需移除 → 限制（环境设计）

## KI-BT-BCW-005：COM-B到干预函数的映射矩阵

```yaml
id: KI-BT-BCW-005
type: 决策矩阵
evidence: T2
stage: S0-S5
domain: behavioral
keywords: [COM-B映射, 干预函数选择, 行为诊断, 决策矩阵]
```

**COM-B诊断结果→推荐干预函数**（基于UCL专家共识）

| COM-B诊断 | 首选干预函数 | 次选干预函数 |
|-----------|------------|------------|
| **心理能力不足**（缺知识/技能） | 教育、训练 | 赋能、示范 |
| **身体能力不足**（体力/生理限制） | 训练、赋能 | 环境重塑 |
| **身体机会不足**（缺工具/时间/场地） | 环境重塑、赋能 | 训练、限制 |
| **社会机会不足**（缺支持/被文化阻碍） | 环境重塑（社会）、示范 | 赋能 |
| **反思性动机不足**（不想做/不相信有效） | 教育、说服、示范 | 激励、赋能 |
| **自动性动机问题**（旧习惯/冲动控制） | 环境重塑（移除触发物）、训练（新习惯） | 限制、激励 |

**关键洞见**：同一干预（如发短信提醒）可以服务不同函数——提醒未服药既是教育（提供信息），也是环境重塑（增加提示），还可以是激励（提示接近奖励）。因此BCW强调用"函数"而非"类型"来描述干预。

**Agent行动规则**：每条行为处方生成前，必须明确标注所使用的干预函数类型，帮助Coach了解干预逻辑而非只看表面措施。

## KI-BT-BCW-006：BCTTv1——93个行为改变技术的16类分组

```yaml
id: KI-BT-BCW-006
type: 行为改变工具分类
evidence: T2
stage: S1-S5
domain: behavioral
keywords: [BCT, 行为改变技术, BCTTv1, 目标设定, 反馈监控, 奖励, 习惯形成]
```

**BCT分类体系**

BCT（Behaviour Change Technique）是干预的最小活性成分——最小化但保留完整作用机制的可重复行动单元。BCTTv1包含93个BCT，由国际专家团队通过"卡片分类"技术组织为16大类。

**16类BCT分组（与行健平台高度相关的前8组）**：

| 分组 | 代表BCT | 平台应用 |
|------|--------|---------|
| **1. 目标与计划** | 行为目标设定(1.1)、结果目标设定(1.3)、行动规划(1.4)、目标复查(1.5) | WellnessPlan模块 |
| **2. 反馈与监控** | 行为反馈(2.2)、行为自我监控(2.3)、结果自我监控(2.4) | CGM数据解读、打卡追踪 |
| **3. 社会支持** | 实践性社会支持(3.1)、情感性社会支持(3.3) | 社群功能、Coach支持 |
| **4. 塑造知识** | 关于行为健康后果的信息(4.1)、关于他人批准的信息(4.4) | 教育内容推送 |
| **5. 自然后果** | 后果显著性(5.2)、预期遗憾(5.8)、关于情绪后果的信息(5.6) | CGM饮食镜子功能 |
| **6. 比较行为** | 示范（行为）(6.1)、社会比较(6.2) | 同类群体案例展示 |
| **7. 联想学习** | 提示/线索(7.1)、习惯养成(7.2)、条件反射(7.3) | 习惯锚定、环境设计提示 |
| **8. 重复与替换** | 行为练习/重复(8.1)、习惯逆转(8.2)、渐进任务(8.7) | 21天习惯模块 |

**BehaviorRx处方中必须标注使用的BCT编号**，便于后续疗效分析与知识迭代。

## KI-BT-BCW-008：APEASE评估标准——干预方案可行性检验

```yaml
id: KI-BT-BCW-008
type: 评估工具
evidence: T2
stage: S1-S4
domain: behavioral
keywords: [APEASE, 干预可行性, 可接受性, 成本效益, 公平性]
```

**APEASE六维评估框架**

任何行为改变干预在最终实施前，必须通过APEASE六维评估：

| 评估维度 | 英文 | 核心问题 | 行健平台应用 |
|---------|------|---------|------------|
| **可负担性** | Affordability | 在可接受预算内是否能覆盖目标人群？ | 功能设计成本/用户愿付费比 |
| **可实操性** | Practicability | 能否按设计通过预期方式执行到目标人群？ | 手机操作复杂度/老人适配 |
| **有效性/成本效益** | Effectiveness & cost-effectiveness | 有证据支持其有效吗？性价比如何？ | 引用BCT文献证据等级 |
| **可接受性** | Acceptability | 相关利益方（用户/医生/机构）觉得合适吗？ | 用户满意度/医生合规认可 |
| **副作用/安全性** | Side-effects/safety | 对目标人群和其他方有无不良影响？ | 安全红线检查 |
| **公平性** | Equity | 干预是否对不同群体公平？会加剧不平等吗？ | 城乡/经济差异适配 |

**傻瓜相机应用**：CoachAgent在向Coach推荐新干预措施时，自动生成APEASE简评报告，帮助Coach快速判断是否采纳。

**铁律**：一项干预如果在"副作用/安全性"维度存在疑问（如激励设计可能引发节食竞争），必须先与Coach确认再推送给用户。

## KI-BT-BCW-009：8步干预设计流程——BCW系统化设计SOP

```yaml
id: KI-BT-BCW-009
type: 设计方法论
evidence: T2
stage: S0-S4
domain: behavioral
keywords: [干预设计, 8步流程, 行为分析, 目标行为选择, 实施计划]
```

**BCW 8步干预设计标准流程**

**第1步：用行为术语定义问题**
- 明确：谁的行为需要改变？是什么行为？在哪里发生？
- 错误：定义健康问题（如"糖尿病控制不好"）
- 正确：定义行为（如"2型糖尿病患者在家庭就餐场景中的高GI主食选择行为"）

**第2步：选择目标行为**
- 生成候选行为长清单，用4个标准排序：①影响力 ②改变难度 ③溢出效应 ④可测量性
- 原则：聚焦1-2个目标行为比分散干预多个行为更有效

**第3步：规范化描述目标行为**（见KI-BT-BCW-003）

**第4步：COM-B行为诊断**（见KI-BT-BCW-002）
- 多源数据：访谈/问卷/文献/观察，从多角度三角验证

**第5步：识别干预函数**（见KI-BT-BCW-004、005）
- 基于COM-B诊断→映射矩阵→筛选适合的干预函数

**第6步：识别政策类别**
- 判断哪些政策类别能支持已选干预函数的实施

**第7步：识别BCT**（见KI-BT-BCW-006、007）
- 从对应干预函数的BCT列表中，基于APEASE标准筛选最优BCT

**第8步：确定内容与实施选项**
- 交付方式：面对面/电话/APP/微信小程序？
- 频次：每天/每周？
- 执行者：Coach/AI Agent/同伴？

**平台SOP应用**：每个新行为处方模块开发都应遵循此8步流程，确保干预逻辑完整。

## KI-BT-BCW-010：BCW与行健平台BehaviorRx的整合应用指南

```yaml
id: KI-BT-BCW-010
type: 平台应用指南
evidence: T2
stage: S1-S4
domain: behavioral
keywords: [BehaviorRx, 行为处方, BCW整合, CoachAgent, 行为改变设计]
```

**BCW框架在行健平台BehaviorRx中的落地设计**

**一、三阶段CoachAgent话术对应BCW逻辑**

| 会话阶段 | BCW对应步骤 | CoachAgent任务 |
|---------|-----------|----------------|
| 发现阶段（S0-S1） | Step 1-3 行为定义与规范化 | 引导用户说清楚"我想在什么情况下改变什么行为" |
| 诊断阶段（S1-S2） | Step 4 COM-B诊断 | 用三维问题识别主要障碍所在 |
| 处方阶段（S2-S3） | Step 5-7 函数+BCT选择 | 生成个性化行为处方并向Coach提交审核 |
| 执行阶段（S3-S4） | Step 8 实施监控 | 追踪BCT执行情况，识别何时需要调整策略 |

**二、BPT-6行为倾向与COM-B的协同诊断**
- **行动型（P1）用户**：COM-B通常缺"计划/反思性动机"→重点用BCT1.4行动规划
