### KI-OPS-001 | 沉浸健康行为养成实践的核心定位

```yaml
knowledge_id: KI-OPS-001
title: 弥合知行分离——沉浸式行为养成项目的核心命题
life_dimension: 项目运营
applicable_stages: [S0, S1, S2, S3]
evidence_tier: T4

retrieval_queries:
  - 沉浸健康行为养成项目是什么
  - 平台线下项目怎么设计
  - 知行分离问题怎么解决
```

**核心命题**

本项目的重点在于**弥合人们在健康素养上的知行分离现象**，养成针对常见病、多发病和自我特定需求的健康生活行为方式。

这直接呼应了 `KI-BH-003`（KAP模式五大局限）：知道≠做到，项目用"沉浸+陪练"取代单纯知识传授。

**与平台理念的映射**

| 项目特征 | 平台技术对应 |
|---------|------------|
| 线上+线下学习 | 小程序 + 社群 + CoachAgent推送 |
| 个性化指导 | BAPS评估 → 个性化行为处方 |
| 支持性氛围营造 | 社群模块 + 打卡系统 |
| 定期评估和调整 | 行为阶段评估 + 处方动态更新 |
| 持续陪练 | CoachPushQueue + Coach工作台 |


**六类优先目标受众**

| 优先级 | 人群类型 | 平台分类 | 逆转潜力 |
|-------|---------|---------|---------|
| 🥇 一 | 五高（高血压/血糖/血脂/尿酸/体重）病后人群 | S2-S4，高风险 | 高 |
| 🥇 二 | 希望远离病痛的人群，**特别是五高人群的亲属** | S0-S2，预防窗口期 | 极高 |
| 🥈 三 | 各类常见疼痛人群，特别是骨伤病后人群 | S3-S4，康复期 | 中高 |
| 🥈 四 | 希望控制体重/减肥的人员 | S1-S3，主动需求 | 高 |
| 🥈 五 | 想戒烟的人群 | S2-S3，已有动机 | 中 |
| 🥉 六 | 糖尿病（60岁以内、肥胖者）和冠心病逆转期人群 | S4-S5，逆转重点 | 极高（T2DM+肥胖） |

**招募优先级逻辑**：
- 优先招募"五高亲属"（二类）：干预成本低、动机强（担心自己"步后尘"），是传播涟漪效应最强的切入口
- 优先招募"T2DM+肥胖60岁以内"（六类）：逆转证据最充分，效果最具示范价值

### KI-OPS-004 | 行为养成四大原则

```yaml
knowledge_id: KI-OPS-004
title: 沉浸健康行为养成的循序渐进、自我激励、社会支持、定期评估四原则
life_dimension: 行为改变理论
applicable_stages: [S1, S2, S3, S4, S5]
evidence_tier: T4
tags: [循序渐进, 自我激励, 社会支持, 定期评估, 行为养成原则]
retrieval_queries:
  - 行为养成有什么原则
  - 怎么帮助用户坚持健康习惯
  - 慢病管理的基本原则是什么
```

**四大原则及平台实现**

**原则1 · 循序渐进**
> 从简单易行的行为开始，逐步增加难度和复杂度

| TTM阶段 | 行为难度要求 | 处方设计要点 |
|---------|-----------|------------|
| S0-S1 | 零行动成本（只需知道/意识到） | 推送知识，不要求行动 |
| S2-S3 | 微习惯（<5分钟/天） | 1个行为，1步操作 |
| S4 | 小目标（15-30分钟/天） | 行为链设计 |
| S5-S6 | 系统整合 | 生活方式全面优化 |

**原则2 · 自我激励**
> 通过设定目标、奖励自己等方式，提高自律意识

平台实现：
- 打卡系统 + 徽章/积分
- 可视化进度（行为阶段爬楼梯图）
- 个人健康数据改善可视化（血糖TIR趋势、体重曲线）

**原则3 · 社会支持**
> 借助家人、朋友、同事等社会力量，共同营造健康生活环境

平台实现：
- 家庭成员绑定功能（家庭视图）
- 社群打卡分享
- Coach社群活跃度管理

**原则4 · 定期评估**
> 定期检查健康状况，调整行为计划，建立良性循环

平台实现：
- 月度行为阶段评估
- 季度健康数据复盘（Coach工作台）
- 处方自动更新建议（MetabolicAgent）

### KI-OPS-005 | 合作方分工模型（可复制推广框架）

```yaml
knowledge_id: KI-OPS-005
title: 沉浸行为养成项目发起方-落地方-专业支持三方分工模型

  - 行为健康项目的商业模式
  - 基层医疗如何参与慢病管理
```

**三方分工清单**

**发起方（中国老年保健协会人文科技健康分会）**
- 项目设计
- 专业专家支持
- 合作方参与人员培训
- 产品支持方协调

**落地合作方（社区卫生服务中心/乡镇卫生院）**
- 参与人员选拔推荐
- 场地提供（≥60平米）
- 入组干预人群选择推荐
- 日常活动的组织和实施
- 参与干预群组的日常维护答疑
- 与专业专家相关的费用支持

**合作方收获（价值主张）**
- 提升工作绩效
- 提升公众认可度
- 能满足以行为养成为目的的良性循环队伍
- 摸索健康行为养成的有效路径
- 打造可复制能推广的行为健康促进模式

**平台数字化映射**
> 本三方分工模型对应平台的**租户（Tenant）架构**：
> - 发起方 = 平台方（行健）
> - 落地合作方 = Tenant（社区/医院）
> - 专业专家 = 行诊智伴(XZB)专家 + 真人Coach

## 铁律声明

**OPS-铁律-01（规模控制）**
线下沉浸项目每期参与人数建议控制在20-50人，超过50人须分组管理，每组配置独立Coach。人多群杂是行为养成项目失败的首要原因。

**OPS-铁律-02（Coach专业门槛）**
区域内医护人员在担任"行为健康促进教练"前，须完成平台规定的基础培训（至少涵盖BPT6分型、TTM阶段识别、5步技能路径）。未培训人员不得独立操作CoachAgent工作台。

**OPS-铁律-03（效果评估不可省略）**
每期项目必须收集效果数据（行为阶段前后变化 + 至少1项健康指标），无效果数据的项目不得作为"成功案例"推广。
