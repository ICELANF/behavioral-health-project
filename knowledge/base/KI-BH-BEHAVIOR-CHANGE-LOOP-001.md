## 核心知识内容

### 1. 三个核心命题

从《行为改变的科学与艺术》课程的核心考察点提炼：

**命题一：评估的真正目标**
> 评估阶段最核心的目标：**识别个体所处改变阶段与关键障碍**（而非判断依从性，不是做诊断结论，不是制定随访计划）

这意味着：
- 评估不是测试，而是理解
- 重点是"这个人现在在哪里"（TTM阶段）而非"他应该在哪里"
- 障碍识别必须个体化：同样是S3（准备期），不同人的关键障碍可能完全不同

**命题二：科学与艺术的整合**
> 行为改变既是科学也是艺术的含义：**在循证框架下进行个体化沟通与策略调整**

- 科学（循证）：提供结构化框架（TTM、SDT、BCT、MI）
- 艺术（个体化）：在框架内灵活响应每个人的独特性
- 不是：所有患者统一流程（过度科学化）
- 不是：完全依靠直觉和意志力（过度艺术化）

**命题三：从评估到行动失败的关键原因**
> 最容易导致失败的关键问题：**缺乏对患者内在动机和现实约束的理解**

- "内在动机"：用户真正在意的是什么（不是医生认为重要的）
- "现实约束"：用户生活中真实的障碍（不是理论上的阻力）
- 行动方案脱离这两者→看起来完美，实际零执行

### 2. 行为改变闭环：五阶段完整模型

```
┌─────────────────────────────────────────────────────────┐
│                    行为改变闭环                           │
│                                                         │
│  ①评估  →  ②理解  →  ③激活  →  ④行动  →  ⑤维持      │
│     ↑                                          ↓        │
│     └──────────── ⑥复盘与再评估 ←─────────────┘        │
└─────────────────────────────────────────────────────────┘
```

#### 阶段一：评估（Assessment）

**目标**：识别所处阶段 + 映射关键障碍

**工具清单**：
```
TTM阶段评估：用户当前意向状态
  → S0：不知道/不认为有问题
  → S1：知道问题，但未考虑改变
  → S2：考虑改变，但未行动
  → S3：准备中，有具体计划
  → S4：正在改变（<6个月）
  → S5：已维持（>6个月）

障碍映射（按类别）：
  □ 知识障碍：不知道怎么做
  □ 技能障碍：知道但缺乏执行技能
  □ 动机障碍：知道且能做，但不想做
  □ 环境障碍：想做但环境不允许
  □ 情绪障碍：情绪状态阻断行动
  □ 关系障碍：社会支持缺失或负向影响
  □ 信念障碍：脚本信念/习得性无助
```

#### 阶段二：理解（Understanding）

**目标**：挖掘内在动机 + 理解现实约束

**核心提问框架**（动机访谈精髓）：
```
挖掘内在动机：
  "对你来说，健康最重要的意义是什么？"
  "如果这个问题改善了，你的生活会有什么不同？"
  "你最在乎的人，希望你做出什么改变？"

理解现实约束：
  "你上次尝试改变的时候，遇到了什么障碍？"
  "你的日常生活中，什么时间最难坚持？"
  "有没有哪些资源，是你感觉自己缺少的？"

整合性提问（动机×约束）：
  "考虑到你现在的生活，你觉得什么样的改变是你真正愿意尝试的？"
```

#### 阶段三：激活（Activation）

**目标**：建立改变信心 + 激活行动意愿

**策略选择（按TTM阶段）**：

| TTM阶段 | 激活策略重点 |
|--------|-----------|
| S0-S1 | 意识唤醒（提供与动机相关的信息），激发矛盾认知 |
| S2 | 动机探索（扩大改变收益），减少改变阻力 |
| S3 | 自我效能建立，资源识别，小目标设计 |
| S4 | 应对策略，预防复发计划 |
| S5 | 意义深化，身份认同强化 |

**自我效能四来源激活**（班杜拉）：
```
成功经验回顾："你有没有曾经坚持做一件事的经历？那次你是怎么做到的？"
替代性学习："你认识有没有和你情况相似，但做到了改变的人？"
言语说服："基于我对你的了解，我认为你有这个能力，因为……"
情绪唤起：帮助用户连接到改变成功后的积极情绪画面
```

#### 阶段四：行动计划（Action Planning）

**目标**：将意愿转化为可执行的行为处方

**SMART行动计划原则**（但需适配个体约束）：
```
具体化（Specific）：不是"多运动"→"周一/三/五午休步行20分钟"
可测量（Measurable）：有可追踪的数字或行为标志
可实现（Achievable）：在用户现实约束内可执行
相关性（Relevant）：连接到用户真正在意的动机
时间性（Time-bound）：有明确的开始时间和第一步

+ 复发预案（非SMART原版内容但BHP必须）：
  "如果某天没完成，你打算怎么办？"
  "什么情况最可能打断你？那时候你能做的最小行动是什么？"
```

#### 阶段五：维持（Maintenance）

**目标**：巩固行为，防止复发，深化意义

**关键机制**：
- 习惯自动化：重复→触发-行为-奖励回路建立
- 身份认同迁移：从"我在尝试改变"→"我就是这样的人"
- 社会环境强化：获得稳定的正向安抚网络
- 成就感积累：小赢策略持续提供正强化

#### 阶段六：复盘再评估（Review）

**目标**：从执行结果中学习，调整下一轮计划

**正确的复盘框架**：
```
✅ 归因正确：
   "这次没完成，是哪个环节出了问题？"（过程分析）
   "什么条件下你做到了？能复制吗？"（成功提炼）

❌ 归因错误：
   "你上周又没做到，是不是不够认真？"（人格归因）
   "看来你对这件事的承诺还不够强"（内部稳定归因）
```

### 3. 科学与艺术的整合操作原则

**科学层（循证框架）**：

| 框架 | 功能 |
|------|------|
| TTM分期 | 识别干预时机，避免阶段错配 |
| SDT三需要 | 理解动机内化层次 |
| BCTTv1 | 选择证据支持的具体行为改变技术 |
| MI动机访谈 | 激活内在动机，减少阻抗 |
| 行为经济学 | 设计环境助推，降低摩擦 |

**艺术层（个体化响应）**：

| 艺术能力 | 表现 |
|---------|------|
| 时机感知 | 何时提供信息，何时保持沉默 |
| 情绪共鸣 | 感知用户情绪状态并匹配响应方式 |
| 语言弹性 | 用用户自己的词汇和隐喻框架 |
| 目标调整 | 在框架内灵活调整计划复杂度 |
| 关系维护 | 在困难对话中维持信任关系 |

## BHP平台应用映射

### 评估模块设计原则

```
BHP评估不是问卷，而是对话：

BAPS评估的正确使用方式：
  ① 先通过BAPS得到结构性数据（科学层）
  ② 用评估结果作为对话的起点，而不是终点（艺术层）
  ③ 在对话中确认、补充和修正评估结果
  ④ 最终以用户自己描述的动机和障碍为准

关键原则：
  → 评估是为了理解，不是为了分类
  → 障碍识别比阶段识别更重要
  → 内在动机必须用户自己说出来（而非被告知）
```

### 行为处方生成的完整闭环

```
行为处方生成流程（BHP平台实现）：

Step 1: BAPS评估 → TTM阶段 + 障碍类型 + 动机维度
Step 2: AI初稿生成 → 基于分期的结构化建议方案
Step 3: 教练对话 → 挖掘内在动机 + 确认现实约束
Step 4: 方案个性化 → 调整目标设定 + 添加复发预案
Step 5: 用户承诺 → 公开承诺+社群见证（利用承诺效应）
Step 6: 执行跟踪 → 打卡+正向安抚+弱信号监测
Step 7: 复盘调整 → 基于执行结果的迭代优化
```
