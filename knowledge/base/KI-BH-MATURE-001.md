### 模块1：情境识别

**TTM阶段范围**：全阶段（S0-S5），不同程度对应

**核心价值**：提供了理解"为什么用户无法自主改变"的心理发展框架，
以及"为什么平台要从约束走向赋权"的理论基础。

**触发关键词**：
`"我就是改不了" / "我需要有人管我" / "我听话的" / "你说怎么做我就怎么做" / "我不知道该怎么选" / "没有动力" / "感觉被控制"`

---

### 模块2：干预核心逻辑

#### 阿吉里斯成熟度七维度 × BHP平台解读

| 发展维度 | 不成熟端（平台用户的起点） | 成熟端（平台帮助用户到达的方向） |
|---------|----------------------|------------------------------|
| 1. 被动→主动 | 等待被提醒、被告知 | 主动记录、主动分析、主动寻求支持 |
| 2. 依赖→相对独立 | 完全依赖医生/教练指令 | 能做有据可循的自我决策，但在需要时求助 |
| 3. 有限→多样行为 | 只会「不吃XX」「多走走」 | 能灵活应对不同场景（应酬/旅行/情绪压力）|
| 4. 浅短→深持兴趣 | 刷到就做，刷不到就忘 | 深入理解自己的行为模式，有长期视角 |
| 5. 短期→长远打算 | 只关注下次体检结果 | 把健康管理嵌入人生规划 |
| 6. 从属→平等/支配 | 「医生让我怎么做我就怎么做」 | 「我了解自己，我能和医生平等对话」 |
| 7. 无自觉→自觉自制 | 依靠外部约束（提醒/打卡/规则）| 自我监控、自我评估、自我调整 |

**核心洞见（平台应用）**：
TTM阶段的推进（S0→S5）本质上就是上述七个维度上从「不成熟端」向「成熟端」的移动。
平台的角色不是永远当外部约束系统，而是**帮助用户在这七个维度上逐渐成熟**——最终不需要平台也能自我管理。

### 模块3：决策树（6节点）

**节点1：成熟度评估**
- 触发条件：用户入档首周，或行为阶段评估时
- Agent话术：
  > "如果明天我消失了，你会不会继续做你现在在做的这些健康管理？"
  > "你对自己的身体，了解到什么程度？你能不能在没有我帮你的情况下，自己判断哪天吃的好、哪天吃的不好？"
- 教练行动：评估用户在七维度中最薄弱的2-3个维度，作为阶段性成长目标

**节点2：被动→主动的干预**
- 触发条件：用户只在收到推送时才行动，从不主动记录或反馈
- Agent话术：
  > "我想请你做一件事——今天吃了什么，不用等我问，你自己先记一下发给我。不需要完美，就是今天你吃的。"
  > "明天早上你醒来，先发我一个字——'起了'。这是你自己的主动动作，不是我提醒你的。"
- 教练行动：设计小的「自发动作」，从等待被激活转向主动参与

**节点3：依赖→相对独立的干预**
- 触发条件：用户对所有决策都询问教练，无法自主判断
- Agent话术：
  > "这个情况你觉得怎么处理比较好？先说说你的想法，然后我们一起看。"
  > "你其实已经知道答案了——你只是在问我确认。我觉得你说的对，你来决定。"
- 教练行动：逐步撤出直接给答案，改为引导用户做判断；肯定用户的判断能力

**节点4：短期→长远视角的建立**
- 触发条件：用户只关注近期指标，对长期改变缺乏兴趣或想象
- Agent话术：
  > "五年后你想要什么样的健康状态？不是具体数字，是一种感觉——你能想象吗？"
  > "你现在做的这件事，三年后对你意味着什么？"
- 教练行动：引导建立长期健康愿景，将当前行为连接到远期自我

**节点5：无自觉→自觉自制的发展**
- 触发条件：用户开始出现主动复盘行为，或询问「我这样对吗」
- Agent话术：
  > "你刚才自己判断了——你觉得这周比上周好还是差？你是怎么判断的？"
  > "你现在能不能自己感觉到，哪天状态比较好、哪天做得不够？"
- 教练行动：强化自我监控能力，肯定用户的自我评估，建立内部参照系

**节点6：成熟度里程碑确认**
- 触发条件：用户展现出多个维度的成熟信号（主动记录/自主决策/长远视角）
- Agent话术：
  > "你有没有注意到，你现在问我问题的方式变了——你不再问『我应该怎么做』，你开始问『我这样想对不对』。这是一个很重要的变化。"
  > "你现在管理自己健康的方式，已经不需要我在旁边了。这是你自己建立起来的。"
- 教练行动：正式确认用户的自主性成长，将教练角色从「指导者」转为「见证者」

### 模块4：M-Action清单

**P1 高优先行动**：
- 在S3-S4阶段主动评估用户的「主动性」和「独立决策」维度
  - 机制：这两个维度是内在动机萌芽的先决条件，也是S4→S5转折的关键信号

- 设计渐进式撤出支持计划（scaffolding fade）：从密集提醒→稀疏提醒→用户自发
  - 机制：与阿吉里斯成熟度发展同步，支持结构随用户成熟度降低

**P2 次优先行动**：
- 对卡在「被动/依赖」端的S4用户，识别是否存在「习得性依赖」（learned helplessness）
  - 机制：长期外部约束会导致部分用户退化出内在动机，需要主动干预

- 将「成熟度维度」纳入教练督导评估（教练有没有帮助用户成长，而不只是「管理用户」）

### 模块5：监测重点（七维度快速评分）

| 成熟度维度 | 评估方式 | 干预触发 |
|-----------|---------|---------|
| 被动→主动 | 自发打卡/记录比例 | <30% → 主动性干预 |
| 依赖→独立 | 自主决策比例（不询问即行动）| <20% → 自主性培养 |
| 有限→多样 | 应对新情境的策略多样性 | 1种策略 → 情境应变训练 |
| 浅短→持久 | 对同一话题的深入持续关注 | 快速跳题 → 深度探索引导 |
| 短期→长远 | 有无3个月以上的目标描述 | 无 → 愿景建立 |
| 从属→平等 | 能否表达自己的不同意见 | 从不表达 → 鼓励自主观点 |
| 无自觉→自觉 | 有无自发复盘和评估 | 无 → 自我监控培养 |

### 模块6：铁律边界

**禁止1**：禁止以「让用户更成熟」为由提前撤出所有支持
- 原因：成熟度是渐进发展的，骤然去除支持等于制造失败体验

**禁止2**：禁止把用户的「依赖行为」解读为「懒」或「不配合」
- 原因：阿吉里斯明确指出依赖状态是发展阶段，不是性格缺陷；贴标签会破坏治疗联盟

**禁止3**：禁止CoachAgent以「你应该更独立」的方式推动成熟
- 原因：成熟度发展需要安全感支撑；施压会产生防御，反而停滞在依赖状态
