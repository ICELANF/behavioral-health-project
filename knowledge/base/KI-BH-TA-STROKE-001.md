## 核心知识内容

### 1. 安抚的定义与生物学基础

**安抚（Stroke）**：任何形式的认可行为——表示"我注意到你的存在"。

伯恩的安抚概念源自哈洛（Harlow）的灵长类研究和斯皮茨（Spitz）的孤儿院研究：**缺乏身体接触和认可的婴儿，即使有基本营养，也会出现身体和心理发育迟缓**。

> 核心命题：**人需要安抚来维持身体和心理的健康**。安抚是人类行为的基本驱动力之一（斯皮茨称之为"刺激饥渴"）。

---

### 2. 安抚的五种类型

| 安抚类型 | 描述 | 示例 | 对行为的影响 |
|---------|------|------|-----------|
| **正面条件安抚** | 因特定行为给予认可 | "你今天完成了步行目标，太好了！" | 强化特定行为（最有效） |
| **正面非条件安抚** | 认可对方的存在本身 | "你是一个很有勇气的人" | 增强自我价值感 |
| **负面条件安抚** | 因特定行为给予否定 | "你这周又没按计划执行" | 可改变行为，但有副作用 |
| **负面非条件安抚** | 否定对方的存在本身 | "你就是没有意志力的人" | 破坏性，损害自我价值 |
| **塑料安抚（虚假安抚）** | 表面正面但不真实 | "你做得真的棒极了！"（当其实没做好） | 长期侵蚀信任关系 |

> **关键原则**：负面条件安抚优于负面非条件安抚（可以批评行为，但不攻击人格）；塑料安抚比负面安抚更具腐蚀性（破坏真实性）。

---

### 3. 安抚经济学（Stroke Economy）——克劳德·斯坦纳的扩展

斯坦纳发现社会传递了五条限制安抚自由流通的"规则"：

| 规则 | 内容 | BHP干预启示 |
|------|------|-----------|
| 不要给予你有的安抚 | "不要随便称赞人，会让对方骄傲" | 教练应打破这条规则，主动给予真实正向安抚 |
| 不要索取你需要的安抚 | "向人求认可是软弱的表现" | 帮助用户开口说出自己的需要 |
| 不要接受你想要的安抚 | "我哪有那么好…" | 教用户真正接受正向反馈（而非deflect） |
| 不要拒绝你不想要的安抚 | "不敢拒绝批评，怕得罪人" | 建立健康的边界能力 |
| 不要给自己安抚 | "自我表扬是自大" | 培养健康的自我关怀 |

> **BHP设计原则**：平台和教练应主动打破安抚经济限制，创造安抚丰盈的环境——真实、及时、具体的正向认可是行为改变持续的基础燃料。

---

### 4. 安抚图（Stroke Profile）与安抚偏好诊断

麦肯纳（McKenna）的安抚图追踪六个安抚维度的使用频率：
- 给予正向安抚 / 给予负向安抚
- 接受正向安抚 / 接受负向安抚
- 拒绝正向安抚 / 拒绝负向安抚

**慢病用户常见安抚图问题**：
```
典型模式一：高负向自我安抚
"我真没用" "我太胖了" "我怎么又失败了"
→ 策略：建立自我慈悲技能+教练增加正面条件安抚

典型模式二：拒绝正向安抚
教练："你这周做得很好！"
用户："哎，也没什么…其实还差得远呢"
→ 策略：训练接受安抚的能力，探索拒绝安抚背后的脚本信念

典型模式三：索取负向安抚
反复"验证"失败叙事，潜意识寻求"对，你就是做不到"的确认
→ 策略：识别心理游戏，不给予负向安抚的"奖励"，重定向到成功体验
```

---

### 5. 自我安抚与内在安抚能力

伯恩认为成熟的人格可以发展出**内部安抚能力**——不依赖外部认可维持自我价值感：
- 健康的自我对话（"我为今天的行动感到满意"）
- 自我关怀实践（休息、身体觉察、情绪承认）
- 从自身行为中提取意义感（而非依赖外部评价）

> 这与SDT的自主性需要高度对应：内部安抚能力发达的人，更能维持内在动机驱动的健康行为。

## BHP平台应用映射

### 6. 安抚系统设计原则

**打卡与反馈的安抚最优设计**：

```
❌ 低效安抚（常见错误）：
   - 千篇一律的"打卡成功！" →无差别非条件式，价值递减
   - 仅追踪数字（运动分钟数）而忽视行为主体 →去人性化安抚
   - 条件安抚只连接结果（"你减了1kg！"）而非行为 →强化错误依赖

✅ 高效安抚（BHP设计）：
   正面条件安抚（聚焦可控行为）：
   "你今天在下雨天依然完成了步行，这份坚持很了不起"

   正面非条件安抚（定期给予）：
   "我注意到你在面对困难时不轻易放弃——你是那种会为自己找到出路的人"

   失败时的保护性安抚（区分行为和人格）：
   "这周没有完成是一个信号，值得我们一起看看发生了什么——这不代表你不行"
```

### 7. 教练会话安抚技术清单

```
每次会话中的安抚清单（教练自查）：
□ 开场时给予至少一个真实的正向条件安抚（基于上周具体行为）
□ 识别并回应用户"拒绝安抚"的行为（不反驳，先探索）
□ 失败复盘时区分行为安抚（条件）和人格安抚（非条件）
□ 会话结束时给予正面非条件安抚（认可其作为人的价值和勇气）
□ 全程避免塑料安抚（虚假的无差别赞美）
□ 帮助用户识别内部负向安抚（自我批评话语），并引导替换
```

### 8. 社群安抚环境设计

```
社群安抚丰盈原则：
- 设置"见证时刻"功能：鼓励同伴认可彼此的努力而非结果
- 展示行为故事而非成绩数字（"他今天在开完三小时会议后还去散步了"）
- 教练定期给予个性化非条件安抚（不依附于任务完成度的关怀）
- 建立"接受安抚练习"：用户学习用"谢谢"替代"哪有什么"
```

## 向量化注记

**主题句**：
TA安抚理论揭示人类对认可的基本需求（安抚饥渴），区分正面条件（强化具体行为）、正面非条件（认可存在价值）、负面条件（纠正行为）和塑料安抚（虚假认可，最具腐蚀性）四类安抚，斯坦纳安抚经济学指出社会规则抑制了安抚的自由流通，BHP教练和平台应通过真实、具体、聚焦行为（而非结果）的正面条件安抚打破安抚经济限制，同时帮助用户识别和替换高频负向自我安抚，建立内部安抚能力以支持内在动机驱动的持续健康行为。
