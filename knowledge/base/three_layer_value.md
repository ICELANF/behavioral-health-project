# 行健平台三层价值架构 | BHP Three-Layer Value Architecture

> 文档版本：v4.0 | 创建：2026-03-07 | 遵循BHP知识库建设规则v4.0 | 行健平台知识库

## 第零层：架构总述

行健平台的核心价值主张由三个递进层次构成，每一层解决不同深度的用户需求，形成从"进门"到"扎根"的完整价值链：

```
L3 成长超越层（为什么留下来）
    ↑ 发现自我 → 实现自我 → 超越自我 → 成为生命的主人
L2 环境适应层（为什么坚持）
    ↑ 建立支持性生活环境 × 社会关系 × 习惯系统
L1 痛苦解决层（为什么进来）
    ↑ 慢病症状缓解 × 风险降低 × 功能恢复
```

**核心命题**：用户因痛苦进来，因环境留下来，因意义而真正改变。

---

## 第一层：痛苦解决层（Pain Resolution）

### 1.1 定义
用户进入平台的首要驱动力是**具体的、可感知的痛苦**：高血糖、肥胖、乏力、慢性疼痛、失眠、情绪低落。这一层的价值在于提供**可信的、可操作的、有效的**症状缓解路径。

### 1.2 核心功能
- **评估**：五维行为模式评估→识别个人化痛苦根源
- **诊断**：BPT-6四维度分型→明确干预靶点
- **处方**：行为处方生成→给出可执行的第一步

### 1.3 适用阶段
- S0无知无觉：尚未意识到自身处于痛苦，需唤醒
- S1矛盾挣扎：知道问题但不知如何解决，需指路
- S2意向萌发：开始考虑改变，需提供具体工具

### 1.4 关键话术模板
> "您现在最困扰您的是___，我们从这里开始。"
> "不需要改变所有，只需要改变这一件事。"

### 1.5 风险提示
- 过度聚焦症状缓解而忽视根源行为→短期改善后反弹
- 处方过于复杂→用户放弃
- **红线**：症状缓解不等于疾病逆转，需持续追踪生物标志物

---

## 第二层：环境适应层（Environmental Adaptation）

### 2.1 定义
痛苦缓解后，用户面临的核心挑战是**维持改变**。这一层的价值在于帮助用户重塑生活环境——物理环境、社会关系、日常习惯系统——使健康行为变成"默认选项"而非"意志力消耗"。

### 2.2 核心功能
- **习惯建筑**：基于福格行为模型（B=MAP）设计微习惯锚点
- **环境设计**：识别并修改触发不健康行为的环境线索
- **社会支持**：激活同伴督导、教练关系、家庭参与
- **追踪反馈**：CGM/可穿戴数据→可视化进步→强化动机

### 2.3 适用阶段
- S3准备行动：开始做具体准备，需系统支持
- S4持续行动：已在执行，需维持和优化
- S5巩固习惯：习惯稳定，防止复发

### 2.4 关键机制
```yaml
习惯锚点公式: 既有习惯 + 新行为 + 即时奖励
环境设计原则: 
  - 减少摩擦（健康选择更容易获取）
  - 增加摩擦（不健康选择更难触发）
  - 视觉提示（行为线索置于显眼位置）
```

### 2.5 常见卡点
- **身份真空**：用户不知道"新的自己"是谁→需身份重塑介入
- **激励悖论**：外部奖励削弱内在动机→需SDT框架支撑
- **环境反弹**：节假日/出差/压力事件破坏习惯→需应急计划

## 第三层：成长超越层（Growth & Transcendence）

### 3.1 定义
这是平台最深层的价值主张，也是让S5-S6用户留下来的根本原因。这一层不再只关注"不生病"，而是关注**生命意义的发现与实现**：发现自我→实现自我→超越自我→成为生命的主人。

### 3.2 四个台阶

| 台阶 | 核心命题 | 平台承载 |
|------|---------|---------|
| 发现自我 | "我是谁？我的身体想告诉我什么？" | Big Five人格探索、体质评估、价值观澄清 |
| 实现自我 | "我能成为什么样的人？" | 可能的自我可视化、成就追踪、技能建构 |
| 超越自我 | "我能为他人/社会贡献什么？" | 同伴支持者角色、健康影响力传播 |
| 成为主人 | "我的生命由我主导。" | S6阶段自我教练能力、生活方式哲学建构 |

### 3.3 适用阶段
- S5-S6：已建立稳定健康习惯，寻找更深层意义
- S6维持超越：不再需要外部提醒，内在驱动持续成长

### 3.4 关键引用
> "人可以被夺走一切，除了一件事——在任何给定的环境中选择自己态度的自由。" ——维克多·弗兰克尔
> "真正的健康不是没有疾病，而是充满活力地活着。" ——行健平台健康哲学

## 三层整合运营逻辑

### 用户旅程映射

```
S0 → S1 → S2  |  痛苦解决层介入  →  首次价值兑现
S2 → S3 → S4  |  环境适应层建构  →  习惯系统成型
S4 → S5 → S6  |  成长超越层激活  →  意义驱动持续
```

### 平台角色切换

| 阶段 | 平台角色 | 用户感受 |
|------|---------|---------|
| L1介入 | 医疗顾问 | "终于找到了解决方案" |
| L2支撑 | 生活教练 | "有人陪我一起改变" |
| L3引领 | 成长伙伴 | "这里帮我成为更好的自己" |

### 防流失设计原则
1. **L1不留人**：只靠症状缓解的用户，症状消失即流失
2. **L2留人靠关系**：社会联结是最强留存因子
3. **L3留人靠意义**：发现生命意义的用户，主动传播平台
