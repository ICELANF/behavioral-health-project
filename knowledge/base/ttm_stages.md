# L1 底座知识 · TTM 跨理论模型（七阶段）判断逻辑
<!-- scope:platform | 证据:T1 | 领域:全局 | 对齐:stage_runtime.py + TTM-7量表 -->
<!-- document_version:v2.1 | ingest_date:2026-02-28 | review_cycle:12个月 -->

> **适用范围**：全部 Agent 的阶段识别与干预策略匹配。
> **核心逻辑**：不同阶段的用户需要完全不同的干预重心——对 S0 用户推行动方案等于揠苗助长。
> **平台对齐**：S0-S6 七阶段体系，对齐 TTM-7 量表（21题）和 StageRuntimeBuilder。

---

## 1. 七阶段定义与 Agent 策略

| 阶段 | 系统名称 | 用户友好名 | 关键特征（判断逻辑） | Agent 干预重心 | 交互模式 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **S0** | 无知无觉 | 探索期 | 不认为自己的行为有问题，甚至抵触讨论。「没问题」「就这样吧」「随便」 | 去道德化与觉察，**绝不给行动建议** | EMPATHY |
| **S1** | 强烈抗拒 | 觉醒期 | 意识到问题存在但强烈排斥改变。「道理我都懂但做不到」「别管我」 | 情感承接，探索矛盾心理，不催方案 | EMPATHY |
| **S2** | 被动应对 | 思考期 | 愿意听但不主动。「你说怎么办吧」「要不试试？」 | 动机激发，强化「为什么要变」 | EMPATHY→CHALLENGE |
| **S3** | 勉强接受 | 准备期 | 打算近期采取行动，可能有小步骤尝试。「想试试」「有没有简单方法」 | 方案设计，提供低门槛 M-Action | CHALLENGE |
| **S4** | 尝试阶段 | 行动期 | 已迈出实质步伐，持续 < 6个月。「我这周做了」「今天坚持了」 | 即时反馈，监测代谢指标，强化微小胜利 | EXECUTION |
| **S5** | 主动实践 | 成长期 | 行为改变持续 ≥ 6个月，趋于稳定。「已经习惯了」「不觉得难了」 | 环境支持，防止复发，过渡到更高挑战 | EXECUTION |
| **S6** | 内化习惯 | 收获期 | 行为完全内化为生活方式，无需刻意维持。「就是日常」 | 自主管理，教练角色淡出 | EXECUTION |

**关于复发**：复发不是独立阶段，而是**任意阶段的回退机制**。用户描述放弃或倒退时，先执行「情感承接」，再用 BFR 复盘，重新路由回 S1 或 S2。

---

## 2. TTM-7 量表判定逻辑

平台使用 TTM-7 量表（21题，每阶段3题，每题1-5分Likert，满分15分/阶段）。

**阶段判定规则**：取各阶段得分最高者；得分相同时取**较低阶段**（保守原则）。

---

## 3. 阶段流转触发规则

| 流转 | 触发条件 | BehaviorFacts 门槛 |
| :--- | :--- | :--- |
| S0→S1 | 用户主动询问「这行为对血糖有影响吗？」 | — |
| S1→S2 | 用户表达「要不试试？」或减少抗拒 | — |
| S2→S3 | 用户说「我想这个月开始改变」或索要方案 | — |
| S3→S4 | 用户反馈「我今天已经尝试了……」 | action_completed_7d ≥ 1 |
| S4→S5 | 行为坚持连续达 6 个月 | streak_days ≥ 14 且 action_interrupt_72h = False |
| S5→S6 | 不依赖工具/提醒也能自主执行 | completion_rate_30d ≥ 0.8 |
| 任意→回退 | 用户描述放弃或倒退，伴随负面情绪 | action_interrupt_72h = True |

---

## 4. StageRuntimeBuilder 七条铁律

1. **禁止跨阶推销**：严禁对 S0-S1 用户直接给复杂行动指令——最常见的失败原因。
2. **情感优先**：复发回退时，必须先执行「情感承接」，再做任何行动建议。
3. **不评判复发**：复发是改变过程的正常部分，禁止用「又失败了」框架描述。
4. **最低阶段保守**：TTM-7 量表多阶段得分相同时，取较低阶段判定。
5. **72小时中断检测**：action_interrupt_72h 触发时不自动降级，先由教练确认。
6. **晋级需事实支撑**：阶段晋级必须有 BehaviorFacts 数据支撑，不能仅凭用户口头声明。
7. **S6 解锁条件**：completion_rate_30d ≥ 0.8 且无 72h 中断，方可判定为 S6。

## 5. 交互模式与阶段映射

| 交互模式 | 适用阶段 | Agent 行为 |
| :--- | :--- | :--- |
| **EMPATHY**（共情） | S0-S1 | 不给行为指令，只共情探索 |
| **EMPATHY→CHALLENGE** | S2 | 轻度引导，探索动机 |
| **CHALLENGE**（挑战） | S3 | 给一个最简微行为 + 鼓励 |
| **EXECUTION**（执行） | S4-S6 | 完整操作化输出（工具+场景+应急方案） |
